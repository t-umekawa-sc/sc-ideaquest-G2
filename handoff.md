## アプリのコンセプト（確定）

- **ideaquest = ゲーム感覚で楽しくアイデアを出し・管理できる WEB アプリ**
- テーマ:「ゲーム感覚で楽しく操作できる」ゲーミフィケーション
- 中心概念:「**クエスト**」= アイデア出しのテーマ単位（カテゴリー・目的/テーマ・期限日・参加メンバー=パーティーを設定して作成）
- コア機能:
  - クエスト作成・管理、パーティー編成
  - クエストに紐づくアイデアの登録・管理（CRUD）
    - アイデア項目: 件名 / タイムリミット / 利害関係者 / アイデア / 価値 / 備考・特記事項 / 関連資料ファイル添付
  - アイデアへの**賛成/反対の投票**（メンバー）
  - アイデアごとの**チャットグループ**（メンバー間の意見交換）
  - **評価者による評価**: 新規性・影響度・実現度・適合性・コストを各5点満点で採点＋観点別コメント
  - 評価者＝クエスト作成者＋作成者が権限付与した参加メンバー（クエスト単位の権限、複数可）
  - **クエスト内権限（6種）**: 所有者/クエスト管理/評価者/アイデア投票/アイデア作成/コメント作成
    - 所有者=全操作可・作成者は既定で剥奪不可・所有者付与は作成者のみ。管理権限で他権限の付与/剥奪・メンバー調整が可能。権限対象は参加メンバー限定
  - **経験値(XP)**: アイデア投稿・ログインで獲得 → レベルアップ
  - **コイン**: 獲得 → ショップで装備購入
  - **3D アバター**: 購入した装備を着せ替え・反映

## 確定した要件

- **ターゲットユーザー = 企業**（社内でアイデアを集めたい組織）。操作するのは従業員と管理者。
  - これに伴い README に反映済み: マルチテナント(企業単位分離)、ロール(一般/管理者)、管理者機能、SSO(案)、Company エンティティ追加、ペルソナ追加。

## まだ決まっていないこと（重要な TBD）

- 提供形態: 複数企業が使う SaaS（マルチテナント）か、1社ごとの個別導入か
- 認証方式: 独自アカウントか、企業の SSO（SAML/OIDC）連携か
- アイデアの公開範囲: 社内全体／部署単位／投稿者と管理者のみ
- XP・コインの具体的な付与ルールとレベル設計（何をしたら何ポイントか）
- 装備アイテムの種類・入手方法（購入のみか、報酬でも入手か）
- 3D アバターのモデル形式（glTF/VRM 等）・着せ替えの実現方式
- 技術スタックの確定（フロント3Dは Three.js / React Three Fiber を想定中）
- マネタイズの有無（コイン課金など）
