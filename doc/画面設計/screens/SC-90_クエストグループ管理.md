# SC-90 クエストグループ管理（画面設計 / 機能詳細）

> 画面遷移図: `doc/画面設計/画面遷移図.md` ／ モック: `doc/画面設計/mocks/SC-90_クエストグループ管理.html` ／ 要件: `doc/要件定義/README.md`（FR-17・第2/6節・データモデル）

- 最終更新: 2026-07-19
- ステータス: ドラフト（機能詳細）

## 1. 概要・目的

- **クエストグループ管理者**が、**自分が管理するクエストグループ内のアカウント**を**登録（発行）・編集・論理削除**する画面。
- アカウントは**管理者発行のみ**（自己新規登録なし）。発行後、対象者は**初回ログイン時にパスワード設定**。
- **所属の割当（グループ間の移動）はシステム管理者の領分（SC-91）**。本画面は自グループ範囲のアカウント運用に限定。
- 導線: ダッシュボード（SC-01）のロール条件付き管理導線（クエストグループ管理者のみ表示）。

## 2. アクセス条件・権限

| 項目 | 内容 |
| --- | --- |
| アクセス可能者 | **クエストグループ管理者**（自分が管理するグループのみ） |
| できること | 自グループ内アカウントの**発行 / 編集 / 論理削除（無効化）/ パスワード再設定** |
| できないこと | 他グループへの所属割当・グループ横断操作・会社設定（＝システム管理者 SC-91） |

## 3. レイアウト構成（決定・業務層）

- **共通ヘッダー**（管理者もゲーム層ステータスを持つため Lv/コイン/SP＋ベル＋アバターメニュー）。
- **管理グループの選択**: 管理者が複数グループを管理する場合に切替（`select`）＋メンバー数。
- **ツールバー（一覧標準 `.list-toolbar`）**: 検索（氏名/ID）＋状態フィルタ（有効/無効）＋**「＋アカウント発行」**。
- **アカウント一覧テーブル（`.table`）**: 氏名 / ログインID / グループ内ロール / 状態 / 最終ログイン / 操作。
- **発行/編集モーダル**（モーダル標準）。

## 4. 表示・入力項目

### 4.1 一覧テーブル
- **氏名 / ログインID / メールアドレス / グループ内ロール（一般メンバー / クエストグループ管理者）/ 状態（有効 / 無効）/ 最終ログイン / 操作**。※ログインIDとメールは別項目。
- 操作＝**編集 / パスワード再設定 / 無効化（⇄再有効化）**。

### 4.2 発行/編集モーダル
- **氏名（必須）/ ログインID（必須・ログイン識別子）/ メールアドレス（必須・OTP/初回設定リンク送付先）/ 所属グループ（＝管理グループに固定表示）/ グループ内ロール**。※ログインIDとメールは別項目。
- 発行時の注記＝初回ログイン時にパスワード設定（招待/初回パスワード設定フロー）。

## 5. アクション・遷移

| トリガー | 条件 | 遷移/結果 |
| --- | --- | --- |
| ＋アカウント発行 | — | モーダル→作成。管理グループに追加・状態=有効（未ログイン）。対象者は初回ログインでパスワード設定 |
| 編集 | — | モーダル→氏名/ID/ロールを更新 |
| パスワード再設定 | — | 確認→初回パスワード設定フローに戻す（リンク送付・デモ） |
| 無効化（論理削除） | 有効なアカウント | 確認→**状態=無効**（ログイン不可）。**入力（アイデア/投票/評価/コメント）は削除せず表示継続**。物理削除しない |
| 再有効化 | 無効なアカウント | 状態=有効に戻す |
| 管理グループ切替 / 検索 / 状態フィルタ | — | 一覧を更新 |

## 6. データ・API（想定）

- **GET `/api/admin/groups/{groupId}/accounts`**: 自グループのアカウント一覧（権限チェック＝当該グループの管理者か）。
- **POST `/api/admin/accounts`**: 発行（管理DBに Account 作成＋`AccountQuestGroup` に当該グループを割当・初回パスワード設定トークン発行）。
- **PATCH `/api/admin/accounts/{id}`**: 氏名/ロール更新。
- **POST `/api/admin/accounts/{id}/password-reset`**: 初回パスワード設定フローへ。
- **POST `/api/admin/accounts/{id}/deactivate` / `/activate`**: **論理削除（無効化）/ 再有効化**（`Account` の状態フラグ・データは保持）。
- 権限は**サーバーで管理グループ範囲に限定**（他グループのアカウントは操作不可）。所属割当（グループ変更）は SC-91（システム管理者）。

## 7. アクセシビリティ / 表示配慮

- 状態は色バッジ＋テキスト（有効/無効）。無効行は淡色＋「再有効化」明示。
- 破壊的でない**論理削除**（無効化）＋確認ダイアログで誤操作を防止。
- テーブルはキーボード操作可・操作ボタンにラベル。

## 8. 状態・バリエーション

| 状態 | 表示 |
| --- | --- |
| 管理グループが1つ | グループ切替は単一表示 |
| 無効アカウント | 淡色＋「再有効化」。フィルタ「無効」で抽出 |
| 該当なし | `.list-empty` |
