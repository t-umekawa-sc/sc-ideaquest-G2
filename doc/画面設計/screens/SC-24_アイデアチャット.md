# SC-24 アイデアチャット（画面設計 / 機能詳細）

> 画面遷移図: `doc/画面設計/画面遷移図.md` ／ モック: `doc/画面設計/mocks/SC-24_アイデアチャット.html` ／ 要件: `doc/要件定義/README.md`（チャット・添付・メンション・通知）
> 対象デザイン標準: `doc/画面設計/デザイン標準.md`（業務層＝クリーン）

- 最終更新: 2026-07-18
- ステータス: ドラフト（機能詳細）

## 1. 概要・目的

- **1つのアイデアに紐づくチャット**（Idea ⇔ ChatGroup が 1:1）。パーティー内で議論・ブラッシュアップする場。
- 表示は **フラットリスト（Slack/Teams 風）**＝アバター＋氏名＋本文を横幅で並べる（長文・メンション・添付が読みやすい・SC-22 プレビューと一貫）。
- SC-22 の「チャットを開く」／通知（SC-02）から遷移。**フルページ**（モーダルにはしない）。

## 2. アクセス条件・権限

| 項目 | 内容 |
| --- | --- |
| 閲覧 | **当該クエストのパーティー全員**（アイデアの公開範囲＝パーティー内） |
| 投稿 | **コメント作成権限**（新規参加メンバーに既定付与）。無い場合は入力欄を無効化＋理由表示 |
| 添付 | 投稿権限に含む。実体は MinIO・物理名ハッシュ化・元名は DB 保持 |
| メンション | パーティーメンバーを `@` で指定 → 対象にアプリ内通知 |

## 3. レイアウト構成（決定）

- **上部＝コンパクトな文脈バー**（`.chat-context`）: クエスト名・カテゴリ／アイデア件名／参加人数・件数／「アイデア詳細を開く（SC-22）」／← 戻る。
- **中央＝メッセージスレッド**（`.chat-thread`・フラットリスト）: 日付セパレータ・未読セパレータ・メッセージ（アバター＋氏名＋時刻＋本文＋添付）。
- **下部＝入力欄**（`.composer`・`position: sticky; bottom:0`）: テキストエリア（自動リサイズ）＋ 📎添付 ＋ 送信、`@` メンション候補ポップアップ、補足（Enter送信/Shift+Enter改行・権限・+5XP）。

## 4. 表示項目

### 4.1 文脈バー
| 項目 | 補足 |
| --- | --- |
| クエスト名・カテゴリ | 小さめ（文脈） |
| アイデア件名 | 💬＋件名（1行省略） |
| メタ | 参加人数・コメント件数 |
| アイデア詳細（SC-22） | ボタンで戻る先を明示 |
| ← 戻る | SC-22（呼び出し元） |

### 4.2 メッセージスレッド
| 項目 | 種別 | 補足 |
| --- | --- | --- |
| 日付セパレータ | `.chat-day` | 日付が変わる区切り |
| 未読セパレータ | `.unread-sep` | 「ここから未読」。最終既読位置に表示 |
| アバター/氏名 | `.avatar sm` ＋名前 | **自分の投稿（`.msg.is-me`）は淡い色地＋左アクセント帯＋名前を強調**して見分けやすくする（フラットリストは維持）。加えて「（あなた）」タグ |
| 時刻 | テキスト | HH:MM |
| 本文 | テキスト | 改行・長文対応。`@氏名` を強調（**自分宛メンションは警告色でさらに強調**）。編集済みは「（編集済み）」タグ |
| 引用返信 | `.msg__quote` | 返信元メッセージの発言者＋抜粋を本文の上に表示 |
| 添付（ファイル） | `.file-chip` | ファイル名＋サイズ＋ダウンロード（署名付きURL） |
| 添付（画像） | `.msg__gallery`/`.msg__thumb` | **画像はインラインでサムネイル表示**。複数はギャラリー。クリックで**ライトボックス拡大**（`.lightbox`） |
| ホバーアクション | `.msg__actions` | メッセージにホバーで表示 → 下記 4.2b |

### 4.2b メッセージのホバーアクション
| アクション | 対象 | 挙動 |
| --- | --- | --- |
| 🙂 リアクション | 全員 | リアクションピッカーを開く |
| 💬 引用返信 | 全員 | 入力欄上に返信コンテキストを表示、送信で引用ブロック付きメッセージ |
| ✏️ 編集 | **自分の投稿のみ** | 本文をインライン編集。**下部コンポーザーと同等の入力**＝📎ファイル添付 / `@`メンション（**候補オートコンプリート付き**）/ 😀絵文字 / 太字 / コード / 🔗リンク＋簡易書式（保存/キャンセル）。保存で「（編集済み）」表示・追加した添付も反映 |
| 🗑 削除 | **自分の投稿のみ**（＋管理権限は TBD） | 確認後にトゥームストーン化（「このメッセージは削除されました」）。添付/リアクションは除去 |
> 編集/削除の可否・保持期間・履歴表示の詳細は第9節 TBD。

### 4.3b リアクション（通常＋魔法）
- 各メッセージ下部に**リアクションバー**（`.reaction-bar`）。**＋**（`.reaction-add`）からピッカーを開く。
- **2層**: ①**通常リアクション**（絵文字 👍❤️😄🎉🙏👀・誰でも無料・複数可・エフェクトなし）②**魔法リアクション**（SC-32 で SP 解放済みの魔法・**エフェクトあり・各魔法1チャット1回**）。
- リアクションは**集計チップ**（絵文字/アイコン＋数）。自分が付けた分は `.is-me` で強調。魔法チップは `.magic`（レア色の光る縁）。**ホバーで「誰が押したか」を表示**（本番はポップオーバー、モックは title 属性）。
- **魔法エフェクト**は対象メッセージ枠に**永続付与**（例: 炎＝`spell-fx--fire` で枠が燃える。ほか 氷/雷/キラキラ）。`prefers-reduced-motion` で動きを止め静的な色味のみ。
- **各魔法は1アイデアチャットにつき1回のみ**（炎・氷・雷…をそれぞれ1回ずつ。同じ魔法は2回不可）。使用中の魔法はピッカーで disabled（「使用中」表示）。**魔法チップをクリックで取消**すれば、同チャット内の別メッセージへ付け替えできる（通常リアクション同様の取消）。
- **1メッセージ1魔法（決定・2026-07-19）**: **1つのメッセージに付けられる魔法は1個まで**（種類を問わず）。**他ユーザーが既に魔法を付けているメッセージには付けられない**（早い者勝ち）。既に魔法があるメッセージでピッカーを開くと魔法欄は選択不可（「既に魔法が付いています」の注記）。自分の魔法はチップから取消可・他人の魔法は取消不可。**通常リアクション（絵文字）はこの制限対象外**（複数・複数人可）。
- リアクション・魔法は**装飾/社交演出のみ**（XP・評価・投票に非影響）。

### 4.3 入力欄（composer）
- **構成（決定）**: 上部バー（ⓘ使い方 / ⌄最小化）→ 引用コンテキスト/添付プレビュー → **全幅テキストエリア** → **ツールバー（左＝書式/アクション、右＝送信）**。書式や添付を全幅入力の下にまとめ、ボタンの重なりを解消。

| 項目 | 補足 |
| --- | --- |
| テキストエリア | **全幅・1行**から自動リサイズ（最大 200px）。`@` 入力でメンション候補 |
| メンション候補 | パーティーメンバーを部分一致で絞り込み。↑↓/Enter/Tab で選択・Esc で閉じる |
| ツールバー（左） | 📎添付 ・ `@`メンション挿入 ・ 😀絵文字（本文へ挿入）・ **太字**（`**`）・ コード（`` ` ``）・ 🔗リンク（`[text](url)`） |
| **簡易書式（リッチ入力）** | 本文は **太字 `**` / コード `` ` `` / リンク `[label](url)` / メンション `@氏名`** を装飾表示（Markdown ライトの最小セット） |
| 😀 絵文字 | ボタンでピッカーを開き、カーソル位置に絵文字を挿入 |
| 📎 添付 | 添付予定をチップ表示（×で取消）。複数可 |
| 送信 | Enter 送信 / Shift+Enter 改行。**空メッセージ（本文も添付も無し）は送信不可＝ボタン無効化** |
| 引用コンテキスト | 引用返信中は入力欄上に返信先を表示（✕ で解除） |
| **最小化** | 右上の「⌄ 最小化」で入力欄を**折りたたみ**、スリムバー（「＋ メッセージを入力…」）だけ表示。クリックで再展開。スレッドの縦領域を広く使える |
| 補足（使い方） | 既定は非表示。入力欄上の **ⓘ 使い方** アイコンで開閉（権限・書式・+5 XP・添付/メンション等の説明） |

## 5. アクション・遷移

| トリガー | 条件 | 遷移/結果 |
| --- | --- | --- |
| 送信 | コメント権限 | メッセージ追加・+5 XP（日次上限）・メンション対象へ通知・末尾へスクロール |
| `@` 入力 | — | メンション候補ポップアップ（メンバー絞り込み） |
| 📎 添付 | コメント権限 | ファイル選択（MinIO へアップロード・権限検証） |
| 添付ダウンロード | パーティー | 署名付きURLで取得 |
| アイデア詳細を開く / ← 戻る | — | **SC-22 アイデア詳細** |
| メンション/通知から流入 | — | 該当メッセージ位置へスクロール（SC-02 通知経由） |
| ＋ リアクション | パーティー | ピッカー表示（通常絵文字＋解放済み魔法） |
| 通常リアクション選択 | パーティー | 集計チップを増減（自分の分をトグル） |
| 魔法リアクション発動 | 魔法を解放済み・**その魔法がこのチャットで未使用**・**対象メッセージに魔法が未付与** | メッセージ枠にエフェクト付与＋魔法チップ。**各魔法1チャット1回**＋**1メッセージ1魔法**（他ユーザが付与済みなら不可）。**魔法の解放は SC-32** |
| 魔法リアクション取消 | 発動した本人 | 魔法チップをクリック → エフェクト/チップ除去。その魔法を再び選べる（別メッセージへ付け替え可） |
| 引用返信 | パーティー | 入力欄上に返信コンテキスト表示 → 送信で引用ブロック付き投稿 |
| 編集（自分） | 作成者本人 | 本文をインライン編集・保存で「（編集済み）」 |
| 削除（自分） | 作成者本人（管理権限は TBD） | 確認後トゥームストーン化 |
| 画像サムネイル | パーティー | クリックでライトボックス拡大 |
| 入力欄の最小化/展開 | — | 折りたたみバー ⇄ 通常表示 |

## 6. 状態・バリエーション

| 状態 | 表示 |
| --- | --- |
| コメント権限なし | 入力欄を無効化＋「投稿するにはコメント権限が必要です」 |
| 未読あり | 未読セパレータを表示・初期スクロールを未読先頭へ |

| 自分の投稿 | 「（あなた）」タグ。ホバーで ✏️編集・🗑削除 |
| 編集済み | 「（編集済み）」タグ |
| 削除済み | トゥームストーン（「このメッセージは削除されました」） |
| 入力欄 最小化 | スリムバー「＋ メッセージを入力…」のみ表示（クリックで展開） |

## 7. データ・API（想定）

- **GET `/api/ideas/{id}/chat?before=&limit=`**: メッセージ一覧（投稿者/本文/時刻/添付/メンション）。ページング（過去へ遡上）。既読位置。
- **POST `/api/ideas/{id}/chat`** `{text, attachments[], mentions[]}`: 投稿。+5 XP（初回・日次上限）。メンション対象へ Notification 生成。
- **POST `/api/uploads`**（共通）: 添付アップロード（MinIO・ハッシュ名・元名/サイズ/MIME 保存）。
- **POST `/api/ideas/{id}/chat/read`**: 既読位置更新（未読セパレータ用）。
- 将来: リアルタイム配信（WebSocket / SSE）。MVP はポーリング/再取得でも可（TBD）。

## 8. アクセシビリティ / 表示配慮

- 入力欄・候補は role/aria（`listbox`/`option`・`aria-modal` 不要）。候補はキーボード操作対応。
- メンションは色のみに依存しない（`@` 記号＋文字装飾）。
- `prefers-reduced-motion` 時はスクロールアニメーションを抑制。
