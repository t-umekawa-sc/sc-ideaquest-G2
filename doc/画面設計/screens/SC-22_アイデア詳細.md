# SC-22 アイデア詳細（画面設計 / 機能詳細）

> 画面遷移図: `doc/画面設計/画面遷移図.md` ／ モック: `doc/画面設計/mocks/SC-22_アイデア詳細.html` ／ 要件: `doc/要件定義/README.md`（アイデア・投票・評価・チャット・フォロー）
> 対象デザイン標準: `doc/画面設計/デザイン標準.md`（業務層＝クリーン。詳細は一覧/ダッシュボードから開く際モーダル）

- 最終更新: 2026-07-18
- ステータス: ドラフト（機能詳細）

## 1. 概要・目的

- 1つのアイデアの詳細画面。**本文・価値・添付・投票・評価結果・チャット導線**を集約し、議論と評価の中心になる。
- 表示のみでなく**投票（賛成/反対）はインラインアクション**として本画面で完結。評価は評価者権限で SC-25 へ、議論の本体は SC-24 へ。
- **一覧（SC-12）／ダッシュボード（フォロー・未投票パネル）から開く際はモーダル差し込み**、URL 直/リロードは本フルページ（UI標準）。

## 2. アクセス条件・権限

| 項目 | 内容 |
| --- | --- |
| アクセス可能者 | **当該クエストのパーティーメンバー**（アイデアの公開範囲＝パーティー内のみ）。非メンバーはアクセス不可 |
| 表示 | パーティーなら全員が閲覧可 |
| 投票 | **アイデア投票権限**（既定でパーティーに付与）。1人1票・締切まで変更可・自分のアイデアにも投票可 |
| チャット投稿 | **コメント作成権限**（既定でパーティーに付与）。閲覧はパーティー全員 |
| 編集 | **作成者本人 or クエスト管理権限**。SC-21 編集モーダル |
| 評価 | **評価者権限**のみ「評価する（SC-25）」を表示 |
| 評価結果の閲覧 | **評価者が指定した公開範囲に従う**（既定＝パーティー全員。限定時は投稿者＋評価者＋所有者/管理のみ）→ 第9節 |

## 3. レイアウト構成（決定）

- **ヘッダー（全幅カード）**＝カテゴリ/ステータスバッジ・件名・投稿者（アバター＋Lv＋氏名）・フォロー★・（権限で）編集・メタ（投稿日/更新日/タイムリミット/利害関係者）。
- **メイン＋右レールの2カラム**（`.idea-layout`）。**狭い画面/モーダル/モバイルでは1カラムに縦積み**（SC-01/SC-12 と一貫）。
  - **メイン（左）**: ①アイデア内容（価値／本文／利害関係者）②関連資料（添付）③チャット（直近プレビュー＋SC-24導線）。
  - **右レール**: ①投票 ②評価結果 ③情報（メタ）。
- 業務層のクリーン基調。ゲーム要素はヘッダーの Lv/コイン（共通ヘッダー）と投票XP・評価コインの表記のみ。

## 4. 表示項目

### 4.1 ヘッダー
| 項目 | 種別 | 補足 |
| --- | --- | --- |
| カテゴリー | `.badge badge-muted` | クエストのカテゴリー |
| ステータス | `.badge` | 公開/評価中/選定候補/選定/完了 など（下書きは本画面に出ない＝作成者のみ SC-01） |
| 件名 | 見出し | 必須項目 |
| 投稿者 | `.poster`（`.avatar` ＋氏名） | アバターに Lv 表示・ホバーで氏名 |
| フォロー★ | トグルボタン | ON=ゴールド「★ フォロー中」／OFF「☆ フォロー」。ウォッチ登録（FR-28） |
| 編集 | ボタン(outline) | 作成者本人 or クエスト管理権限のみ表示。**SC-21 編集モーダル** |
| メタ | テキスト | 投稿日 / 更新日 / タイムリミット（近接は警告色）/ 利害関係者 |

### 4.2 メイン: アイデア内容
| 項目 | 補足 |
| --- | --- |
| 価値 | **必須**。このアイデアがもたらす価値 |
| アイデア本文 | **必須**。本文（改行・長文対応） |
| 利害関係者 | 任意 |

### 4.3 メイン: 関連資料（添付）
- ファイル一覧（アイコン＋元ファイル名＋サイズ/アップロード者/日時＋ダウンロード）。0件なら非表示。
- 実体は **MinIO**・物理名はハッシュ化。ダウンロードは**権限検証＋署名付きURL**。上限サイズ/許可形式は TBD（要件）。

### 4.4 メイン: チャット（導線＋直近プレビュー）
- **直近メッセージを数件プレビュー**（アバター＋氏名＋時刻＋本文、`@`メンション強調）＋「**チャットを開く（SC-24）→**」。
- パーティー全員が閲覧・投稿可（コメント作成権限）。添付・メンション・通知対応（本体は SC-24）。
- 0件なら「まだコメントはありません」＋開くボタン。

### 4.5 右レール: 投票
| 項目 | 補足 |
| --- | --- |
| 集計 | ▲賛成 n / ▼反対 n。**会社の匿名化設定に従う**（既定＝匿名＝集計数のみ） |
| 投票ボタン | 賛成 / 反対（自分の投票を `is-on` でハイライト）。**1人1票・締切まで変更可**・再クリックで取消 |
| 記名モード | 会社設定が記名なら賛成/反対の**投票者アバターを並べて表示**（`.avatar-group`）。匿名モードは数のみ |
| 付与 | 投票で **+5 XP**（初期値・日次上限対象）。評価スコアには非影響（投票と評価は独立） |
| 締切後 | サーバー判定で投票不可（ボタン無効化＋理由表示） |

### 4.6 右レール: 評価結果
| 項目 | 補足 |
| --- | --- |
| 公開範囲バッジ | 🔓 パーティー全員 / 🔒 限定（評価者指定・第9節） |
| 総合平均 | 5観点の**均等平均**（/5.0）＋評価者人数 |
| 付与コイン | `round(平均×10)`（最大50）を**投稿者に付与**。表記のみ（付与処理はサーバー） |
| 5観点スコア | 新規性 / 影響度 / 実現度 / 適合性 / **コスト（低コストほど高得点）**。各バー＋数値 |
| **総評** | 評価者ごとの**全体コメント**（`Evaluation.overall_comment`・SC-25 で入力）。評価者アバター＋氏名＋総評テキスト（左アクセント帯付きで観点別コメントと区別） |
| 観点別コメント | 評価者アバター＋氏名＋観点バッジ＋コメント |
| 未評価時 | 「評価待ち」表示（スコア/コメントは非表示） |
| 評価する（SC-25） | **評価者権限のみ表示**。新規/再評価。公開範囲は評価時に指定 |

### 4.7 右レール: 情報（メタ）
- ステータス / カテゴリー / 公開範囲（このクエストのパーティー内）/ 所属クエスト（SC-12へリンク）/ 投稿日 / 最終更新。

## 5. アクション・遷移

| トリガー | 条件 | 遷移/結果 |
| --- | --- | --- |
| 戻る | — | 所属クエスト（SC-12）。モーダル時は閉じる |
| フォロー★ | パーティー | フォローON/OFF（FR-28）。以後の動きを通知 |
| 賛成/反対 | 投票権限・締切前 | 投票登録/変更/取消。集計更新・+5 XP（初回）・匿名化は表示のみ |
| チャットを開く | コメント権限（閲覧は全員） | **SC-24 アイデアチャット** |
| 編集 | 作成者/管理権限 | **SC-21 編集モーダル** |
| 評価する | 評価者権限 | **SC-25 評価画面**（公開範囲指定含む） |
| 添付ダウンロード | パーティー | 署名付きURLで取得 |
| クエスト名 | — | **SC-12 クエスト詳細** |

## 6. 状態・バリエーション

| 状態 | 表示 |
| --- | --- |
| 自分の投票済み | 該当ボタンをハイライト（`is-on`） |
| 未投票 | どちらもニュートラル |
| 締切超過 | 投票/チャット/評価の可否はステータス＋サーバー判定（不可時は無効化＋理由） |
| 評価未実施 | 評価結果＝「評価待ち」 |
| 評価限定公開 | 範囲外ユーザーには結果を「集計のみ」or 非表示（第9節） |
| 添付なし | 関連資料セクション非表示 |
| 自分のアイデア | 編集ボタン表示・自分にも投票可 |

## 7. データ・API（想定）

- **GET `/api/ideas/{id}`**: 件名/価値/本文/利害関係者/タイムリミット/カテゴリ/ステータス/投稿者/日時/添付一覧/自分の投票状態/投票集計（匿名化考慮）/評価集計（公開範囲考慮）/フォロー状態/チャット直近数件。
- **PUT `/api/ideas/{id}/vote`** `{choice: agree|disagree|null}`: 投票登録/変更/取消。+5 XP（初回・日次上限）。
- **PUT `/api/ideas/{id}/follow`** `{on}`: フォロー切替。
- **GET `/api/ideas/{id}/evaluations`**: 公開範囲に応じ、集計＋観点別コメント（範囲外は集計のみ/非表示）。
- **GET `/api/ideas/{id}/attachments/{aid}/url`**: 署名付きダウンロードURL（権限検証）。
- 匿名化（`vote_anonymized`）・管理者開示（`hide_voters_from_managers`）・評価公開範囲はサーバー側で最終制御（クライアント表示だけに依存しない）。

## 8. アクセシビリティ / 表示配慮

- 投票・評価スコアは**色のみに依存しない**（数値＋テキスト＋アイコン）。
- 投票ボタンは `aria-pressed`。フォロー★も `aria-pressed`。
- スコアバーは数値併記。コスト観点は「低コストほど高得点」を明示（ⓘ/ツールチップ）。
- モーダル差し込み時はフォーカストラップ・`aria-modal`・`Esc`/バックドロップで閉じる（モーダル標準）。
