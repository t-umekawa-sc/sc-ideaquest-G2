# SC-02 通知（画面設計 / 機能詳細）

> 画面遷移図: `doc/画面設計/画面遷移図.md` ／ モック: `doc/画面設計/mocks/SC-02_通知一覧.html` ／ 要件: `doc/要件定義/README.md`（FR-24・FR-28・FR-34・第6節「通知」・データモデル Notification）
> 対象デザイン標準: `doc/画面設計/デザイン標準.md`（**業務層**＝クリーン。読ませる一覧）

- 最終更新: 2026-07-19
- ステータス: ドラフト（機能詳細）

## 1. 概要・目的

- **アプリ内通知の一覧**画面。各種イベント（メンション・自アイデアへのコメント・フォロー中アイデアの動き・アイデア更新・実績獲得・魔法リアクション 等）を時系列で表示し、**クリックで参照先へ遷移**する。
- **MVP はアプリ内通知**（ベル/未読）。メール等の外部通知は将来対応（第6節）。
- 導線: **共通ヘッダーの通知ベル（🔔＋未読バッジ）＝全認証画面から**／ダッシュボード（SC-01）「最近の通知」パネルの「すべての通知 →」・ショートカットタイル「通知」。

## 2. アクセス条件・権限

| 項目 | 内容 |
| --- | --- |
| アクセス可能者 | 認証済みユーザー全員（自分宛の通知のみ） |
| 生成 | サーバーがイベント発生時に `Notification` を作成（宛先 User・種別・参照先・既読フラグ） |
| 既読 | 通知を開く（参照先へ遷移）／「既読にする」／「すべて既読にする」で既読化。未読数はヘッダーのベルバッジと同期 |

## 3. レイアウト構成（決定・業務層）

- **共通ヘッダー**（Lv/コイン/SP＋ベル＋アバターメニュー）。本画面ではベルが現在地（`aria-current`）。
- **絞り込み・一括操作（一覧標準 `.list-toolbar`）**: 状態（すべて/未読のみ）＋種別（メンション/コメント/評価/選定/更新/実績/魔法）＋**未読件数**＋**「すべて既読にする」**。
- **通知リスト（日付グループ `今日 / 昨日 / それ以前`）**: 各通知＝種別アイコン＋本文＋文脈（クエスト/アイデア）＋時刻＋未読ドット＋「既読/未読」トグル。**行クリックで参照先へ遷移**。
- **レイヤー方針（1レイヤー1マテリアル）**: 通知は「読ませる情報」＝**業務層のクリーン不透明**（ゲーム層のCRTガラスは使わない）。獲得表示（XP/コイン/SP）だけピクセル色のアクセント。

## 4. 表示・入力項目

### 4.1 絞り込み・一括操作
- **状態**（すべて/未読のみ）／**種別**（メンション/コメント/評価/選定/アイデア更新/実績/魔法）／**未読件数**／**すべて既読にする**。

### 4.2 通知の種別（`Notification.type`・参照先）

| 種別 | アイコン | 例 | 参照先 |
| --- | --- | --- | --- |
| メンション | @ | チャットで `@あなた` | SC-24 アイデアチャット |
| コメント（自アイデア） | 💬 | 自分のアイデアに新規コメント | SC-22 アイデア詳細（チャット） |
| フォロー: 新規コメント | 💬 | フォロー中アイデアに新規コメント | SC-22 |
| フォロー: 評価 | ⭐ | フォロー中アイデアに新評価 | SC-22（評価結果） |
| フォロー: 選定 | 🏆 | 自分/フォロー中アイデアが選定 | SC-22 |
| アイデア更新（差分） | 🔄 | 投票済み/フォロー中アイデアが更新（版追加）→**投票の見直し**導線 | SC-22（更新履歴/差分） |
| 実績獲得 | 🎖️ | 実績獲得＋**ティア連動コイン**（◆20/50/150） | SC-40 実績/バッジ |
| 魔法リアクション | ✨ | 自分のメッセージに魔法が付いた | SC-24 |

- 一部の通知は**獲得表示**（選定＝＋XP・コイン／実績＝コイン）を併記。**アイデア更新**通知は「投票の見直し」タグ付き（FR-34）。

### 4.3 通知アイテム
- 未読は**背景ハイライト＋赤ドット**。既読で通常表示。個別「既読/未読」トグル可。

## 5. アクション・遷移

| トリガー | 条件 | 遷移/結果 |
| --- | --- | --- |
| 通知をクリック | — | 既読化して**参照先へ遷移**（種別ごと・上表） |
| 既読にする / 未読に戻す | — | 当該通知の既読フラグを反転（遷移しない）。未読数・ベルバッジを更新 |
| すべて既読にする | — | 全通知を既読化 |
| 状態/種別で絞り込み | — | 一覧を更新 |
| （ヘッダー）ベル 🔔 | — | 本画面（SC-02）へ（全認証画面から） |
| ダッシュボードへ戻る | — | **SC-01** |

## 6. データ・API（想定）

- **GET `/api/notifications?state=all|unread&type=...`**: 自分宛の通知（種別/本文/参照先/既読/作成日時）。未読数も返す。ページング想定。
- **POST `/api/notifications/{id}/read`**（or `/read-all`）: 既読化。**参照先クリック時にも既読化**（サーバー）。
- 生成はイベント駆動（メンション/コメント/評価/選定/更新〔`IdeaRevision`〕/実績/魔法リアクション）。**フォロー通知は `Follow`**・**更新通知は投票者＋フォロワー**へ（FR-28/FR-34）。
- 外部通知（メール等）は将来（第6節・TBD）。

## 7. アクセシビリティ / 表示配慮

- 未読は色（ハイライト/ドット）だけでなく「既読にする/未読に戻す」ラベル＋未読件数で明示。
- 通知行はリンク（キーボード到達可）。ベルは `aria-label`（未読件数）。
- 業務層のクリーン表示で可読性を確保（背景画像スクリムと整合）。

## 8. 状態・バリエーション

| 状態 | 表示 |
| --- | --- |
| 未読0 | ベルバッジ非表示・「0 件の未読」 |
| 通知なし / 絞り込み該当なし | `.list-empty`（該当する通知はありません） |
| 大量 | 日付グループ＋ページング/無限スクロール（TBD） |
