# SC-11 クエスト作成・編集（画面設計 / 機能詳細）

> 画面遷移図: `doc/画面設計/画面遷移図.md` ／ モック: `doc/画面設計/mocks/SC-11_クエスト作成編集.html` ／ 要件: `doc/要件定義/README.md`（FR-20/21/22/27・第6節「権限モデル」・クエストの入力項目）
> 対象デザイン標準: `doc/画面設計/デザイン標準.md`（業務層＝クリーン。登録・編集はモーダル標準）

- 最終更新: 2026-07-18
- ステータス: ドラフト（機能詳細）

## 1. 概要・目的

- **クエストを新規作成／編集**する画面。カテゴリー・目的/テーマ・期限・クエストグループ・**パーティー（参加メンバー）と権限**を設定する。
- UI 標準に合わせ**独立画面ではなくモーダル**（Next.js Parallel Routes ＋ Intercept Routes・URL 直/リロードはフルページ）。
- 開く導線: **SC-10 クエスト一覧「＋作成」／ SC-12 クエスト詳細「編集」（所有者・管理）／ SC-01 下書きパネル（続きを編集）**。
- 作成者は**既定で所有者**になる。作成で公開しパーティーに通知＋作成 XP を付与、**下書き保存**なら本人のみ表示（付与なし）。

## 2. アクセス条件・権限

| 項目 | 内容 |
| --- | --- |
| 作成 | 認証済みユーザー（作成者＝所有者になる） |
| 編集 | 当該クエストの**所有者・クエスト管理権限者**のみ |
| パーティー候補 | **選択中のクエストグループの所属者に限定**（FR-15） |
| 所有者権限の付与 | **作成者のみ**が他者に付与できる。作成者の所有者権限は**剥奪不可** |
| その他権限の付与/剥奪 | 所有者・クエスト管理権限者が可 |

## 3. レイアウト構成（決定）

- **モーダル**（`.modal` 標準）。パネル内に上から:
  1. **ドラッグ用ヘッダー**（⠿グリップ＋タイトル「クエストを作成／編集」）＋権限注記
  2. **クエストアイコン**（アップロード or 未設定フォールバック プレビュー）
  3. **件名**（必須）
  4. **カテゴリー**（必須・事前定義＋自由入力）／**期限日**（必須）の 2 カラム
  5. **目的・テーマ**（必須・textarea）
  6. **クエストグループ**（必須・select。変更でパーティー候補が入れ替わる）
  7. **パーティー・権限**（メンバー一覧＋権限トグル＋候補から追加）
  8. フッター（クエストを作成／更新・**下書き保存**・キャンセル）
- 業務層のクリーン基調。**ヘッダーをつかんでドラッグ移動可**（SC-25 と同標準）。

## 4. 表示・入力項目

### 4.1 基本項目

| 項目 | 必須 | 入力 | 備考 |
| --- | --- | --- | --- |
| クエストアイコン | 任意 | 画像アップロード | 未設定時は**件名の頭文字＋所有者アバター**を自動描画（`.quest-icon` フォールバック）。実体は MinIO・`Quest.icon_image_path`（FR-20） |
| 件名 | **必須** | text | クエスト名。一覧/詳細/アイコン頭文字に使用（**要件表に未記載だったため追加＝§9 で README 整合を確認**） |
| カテゴリー | **必須** | combobox | **事前定義リスト＋自由入力**（「＋自由入力で追加」で新規カテゴリ名を入力） |
| 期限日 | **必須** | date | アイデア募集/クエストの締切日 |
| 目的・テーマ | **必須** | textarea | 何を達成したいか・どんなアイデアを募るか |
| クエストグループ | **必須** | select | クエストは 1 グループに属す。**パーティー候補を規定**（同一グループ所属者に限定） |

- 件名を入力すると**アイコンのフォールバック頭文字**が連動更新（モックで実演）。

### 4.2 パーティー・権限（FR-22 / FR-27 / 第6節「権限モデル」）

- **6 権限**: 所有者 / クエスト管理 / 評価者 / アイデア投票 / アイデア作成 / コメント作成。各メンバー行で**トグルチップ**として付与/剥奪。
- **作成者行**: 「あなた・作成者」バッジ。**所有者オン固定・全権限は剥奪不可（`aria-disabled`）・パーティーから外せない**。
- **メンバー追加**: 候補（同一グループ所属者）から選んで追加。追加時に**既定権限＝投票・作成・コメント**を自動付与（管理/評価者/所有者はオフ）。
- **所有者チップ**は作成者のみ他者に付与可（権限モデル）。UI では所有者チップをコイン色で区別。
- **メンバー削除（パーティーから外す）**: ✕ で除外し候補へ戻す。**それまでの入力（アイデア/投票/評価/コメント）は削除せず表示継続**（第6節・表示側の扱い）。

## 5. アクション・遷移

| トリガー | 条件 | 遷移/結果 |
| --- | --- | --- |
| アイコンをアップロード/未設定に戻す | — | プレビュー更新（実体は MinIO） |
| 件名を入力 | — | フォールバックアイコンの頭文字を更新 |
| カテゴリー「自由入力で追加」 | — | 新規カテゴリ名を入力して選択 |
| クエストグループ変更 | — | **パーティー候補を入れ替え**（別グループ所属者を除外） |
| 候補メンバーを追加 | 所有者/管理 | 行を追加し既定権限を付与 |
| 権限チップをトグル | 所有者/管理（所有者付与は作成者のみ） | 付与/剥奪 |
| メンバーを外す | 所有者/管理・作成者以外 | パーティーから除外（入力履歴は保持） |
| クエストを作成/更新 | 必須項目充足 | 保存・公開・パーティー通知・**作成時は作成 XP 付与**、閉じる |
| **下書き保存** | — | **本人のみ表示**の下書き（付与なし）。SC-01 下書きパネルから再開可 |
| ヘッダーをドラッグ | — | モーダル移動 |
| キャンセル / ✕ / バックドロップ | — | 破棄して閉じる |

## 6. 状態・バリエーション

| 状態 | 表示 |
| --- | --- |
| 新規作成 | 空フォーム。作成者行のみ（所有者固定） |
| 編集 | 既存値を読み込み「更新」。作成者以外が開けるのは所有者/管理のみ |
| 下書きの続き | 保存済み下書きを読み込み継続編集。公開するまで本人のみ |
| 必須未充足 | 「作成」で警告（件名・カテゴリー・期限日・目的テーマ・メンバー） |
| グループ変更で候補外に | 別グループ所属者は候補から除外（既存メンバーの扱いは §9 TBD） |

## 7. データ・API（想定）

- **POST `/api/quests`** `{name, category, deadline, theme, quest_group_id, icon?, members:[{user_id, permissions[]}], status: draft|open}`: 新規作成。作成者を所有者として保存、`open` で公開＋作成 XP。
- **PUT `/api/quests/{id}`**: 編集（所有者/管理のみ・サーバーで権限検証）。
- **GET `/api/quest-groups/{id}/members`**: パーティー候補（同一グループ所属者）。
- アイコン画像は MinIO へアップロードし `icon_image_path` を保持（物理名ハッシュ化）。
- 権限付与はサーバーで再検証（所有者付与は作成者のみ・既定権限の付与）。

## 8. アクセシビリティ / 表示配慮

- 権限チップは `role="button"`＋`aria-pressed`、キーボード（Enter/Space）操作可。不可の行は `aria-disabled`。
- 必須は `*`（`.req`）＋確定時バリデーション。カテゴリーの自由入力は明示。
- モーダル標準（フォーカストラップ・`aria-modal`・`Esc`/バックドロップで閉じる・`prefers-reduced-motion`）。
