# SC-91 システム管理（運営）（画面設計 / 機能詳細）

> 画面遷移図: `doc/画面設計/画面遷移図.md` ／ モック: `doc/画面設計/mocks/SC-91_システム管理.html` ／ 要件: `doc/要件定義/README.md`（FR-16・第2/8節・データモデル Company/Account/AccountQuestGroup）
> 対象デザイン標準: `doc/画面設計/デザイン標準.md`（**業務層**＝クリーン。トグルは新設 `.switch`）

- 最終更新: 2026-07-19
- ステータス: ドラフト（機能詳細）

## 1. 概要・目的

- **システム管理者（運営）**向けの運用画面。**会社（テナント）の作成・管理**、**会社ごとの設定**（投票匿名化・管理者開示・MFA）、**会社DBのプロビジョニング案内**、**全アカウントとクエストグループ所属のフル操作**を行う。
- 会社DBは物理分離（別コンテナ）。接続情報の実体は `.env`、本画面は**参照キー（DB識別子）**のみ表示。
- 導線: ダッシュボード（SC-01）のロール条件付き管理導線（システム管理者のみ表示）。

## 2. アクセス条件・権限

| 項目 | 内容 |
| --- | --- |
| アクセス可能者 | **システム管理者（運営）のみ** |
| できること | 会社の作成・管理・**会社設定（3フラグ）**・DBプロビジョニング案内・**全アカウント発行/編集・クエストグループ所属の割当（フル操作）** |
| データ配置 | 会社フラグは管理DBの `Company`／アカウント・所属は `Account`・`AccountQuestGroup`（管理DB） |

## 3. レイアウト構成（決定・業務層）

- **共通ヘッダー**（運営バッジ＋ベル＋アバターメニュー。運営はゲーム層ステータスを持たない想定でバッジ表示）。
- **会社（テナント）一覧（`.table`）**: 会社名 / DB識別子 / 状態 / アカウント数 / グループ数 / 作成日 / 設定。行/「設定」で下の会社設定へロード。＋会社を作成。
- **会社設定カード（選択中の会社）**: **`.switch`（新設トグル）で3設定**＋DB識別子＋プロビジョニング注記。
- **アカウント & 所属（一覧標準＋`.table`）**: 会社フィルタ＋検索＋「＋アカウント発行」。氏名 / ログインID / 会社 / システムロール / 所属グループ / 状態 / 操作。
- **モーダル**（会社作成 / アカウント発行・編集）。
- **レイヤー方針**: 運用画面＝**業務層のクリーン**（ゲーム層要素は使わない）。

## 4. 表示・入力項目

### 4.1 会社一覧
- **会社名 / DB識別子（`.env` 参照キー）/ 状態（有効/準備中/停止）/ アカウント数 / グループ数 / 作成日 / 設定**。

### 4.2 会社設定（`.switch` で ON/OFF・Company の各フラグ）

| 設定 | フラグ | 既定 | 補足 |
| --- | --- | --- | --- |
| 投票の匿名化 | `vote_anonymized` | ON | ON=集計数のみ（匿名）／OFF=賛成/反対のアバターを表示（記名） |
| 匿名時に所有者/管理者へ投票者を隠す | `hide_voters_from_managers` | ON | ON=管理者にも非開示／OFF=所有者・クエスト管理者のみ投票者を確認可。**匿名化 OFF（記名）時は無効**（全員に表示） |
| MFA を必須にする | `mfa_required` | ON | ON=ログイン時メールOTP（信頼済み端末はスキップ）／OFF=ID＋パスワードのみ |

- **DB接続識別子**（参照キー）表示＋**プロビジョニング（MVP）注記**（compose 手編集 → `up` → 管理DBに登録）。

### 4.3 アカウント & 所属
- **氏名 / ログインID / 会社 / システムロール（一般 / クエストグループ管理者 / システム管理者）/ 所属クエストグループ（複数可）/ 状態 / 操作**。
- 発行/編集モーダル＝氏名・ログインID・会社・ロール・**所属グループ（`.multiselect`・複数）**。所属割当は**システム管理者のフル操作**。

## 5. アクション・遷移

| トリガー | 条件 | 遷移/結果 |
| --- | --- | --- |
| ＋会社を作成 | — | モーダル→管理DBに Company 登録（状態=準備中）。**会社DBは別途プロビジョニング**（compose）後に有効化 |
| 会社の「設定」/行クリック | — | 会社設定カードへ当該会社のフラグをロード |
| 設定トグル（3種） | — | `Company` のフラグを更新（サーバー保存）。匿名化OFFで「管理者へ隠す」は自動無効 |
| ＋アカウント発行 | — | モーダル→ Account 作成＋所属（`AccountQuestGroup`）割当。初回ログインでパスワード設定 |
| 所属・編集 | — | ロール変更・**所属グループの割当変更**（フル操作） |
| 会社フィルタ / 検索 | — | アカウント一覧を更新 |

## 6. データ・API（想定）

- **GET `/api/sysadmin/companies`**: 会社一覧（名称/DB識別子/状態/件数/フラグ）。
- **POST `/api/sysadmin/companies`**: 会社作成（管理DBに Company 登録・状態=準備中）。**DBプロビジョニングは運用手順**（MVP: compose 手編集＋`.env`）。
- **PATCH `/api/sysadmin/companies/{id}/settings`**: `vote_anonymized` / `hide_voters_from_managers` / `mfa_required` を更新。
- **GET/POST/PATCH `/api/sysadmin/accounts`**: 全アカウントの発行・編集（ロール・所属）。**`AccountQuestGroup`** で所属割当。
- 権限は**システム管理者のみ**（サーバーで厳格化）。運用操作は監査ログ（`SystemAdmin 操作ログ`）に記録。

## 7. アクセシビリティ / 表示配慮

- トグルは `.switch`（ネイティブ checkbox 装飾・focus-visible リング・ON/OFF 状態語併記）。無効時は淡色＋理由文言。
- 状態・ロールは色＋テキスト。DB識別子は等幅表示。破壊的操作は確認。
- 依存関係（匿名化OFF→管理者開示設定は無効）を UI と文言で明示。

## 8. 状態・バリエーション

| 状態 | 表示 |
| --- | --- |
| 会社=準備中 | プロビジョニング未完（DB接続確認後に有効化） |
| 匿名化 OFF（記名） | 「管理者へ隠す」トグルは無効・注記表示 |
| アカウント該当なし | `.list-empty` |
