# SC-91 システム管理（会社一覧）（画面設計 / 機能詳細）

> 画面遷移図: `doc/画面設計/画面遷移図.md` ／ モック: `doc/画面設計/mocks/SC-91_システム管理.html` ／ 要件: `doc/要件定義/README.md`（FR-16・第2/8節・データモデル Company）
> 対象デザイン標準: `doc/画面設計/デザイン標準.md`（**業務層**＝クリーン）
> 関連: **SC-92 会社詳細 / 設定**（会社ごとの設定・アカウント/所属はこちら）

- 最終更新: 2026-07-19
- ステータス: ドラフト（機能詳細）

## 1. 概要・目的

- **システム管理者（運営）**向け。**会社（テナント）の一覧・作成**を行う入口画面。
- 会社ごとの詳細（**設定＝投票匿名化/管理者開示/MFA、アカウント/所属のフル操作**）は、会社を選んで **SC-92 会社詳細** で行う。
- **画面分割の意図**: 「どの会社をメンテしているか」を明確にするため、一覧（SC-91）と会社単位の管理（SC-92）を分離。SC-92 は上部に**メンテ中の会社を示す文脈バナー**を常時表示。
- 導線: ダッシュボード（SC-01）のロール条件付き管理導線（システム管理者のみ）。

## 2. アクセス条件・権限

| 項目 | 内容 |
| --- | --- |
| アクセス可能者 | **システム管理者（運営）のみ** |
| できること | 会社の**一覧・検索・作成**。各会社の管理は SC-92 へ |
| データ | 管理DBの `Company`（名称/会社コード/DB識別子/状態/件数）。接続情報の実体は `.env`（参照キーのみ表示） |

## 3. レイアウト構成（決定・業務層）

- **共通ヘッダー**（運営バッジ＋ベル＋アバターメニュー）。
- **ツールバー（一覧標準 `.list-toolbar`）**: 検索（会社名/会社コード/DB識別子）＋状態フィルタ（有効/準備中/停止）＋件数、**「＋会社を作成」**。
- **会社一覧テーブル（`.table`）**: 会社名（**会社アイコン〔`.quest-icon`＝画像 or 頭文字タイル・会社カラー〕**＋名称）/ **会社コード** / DB識別子 / 状態 / アカウント数 / グループ数 / 作成日 / **「管理する →」（SC-92 へ）**。行クリックでも SC-92 へ。
- **会社作成モーダル**（会社名・**会社コード〔必須・全社一意・英大文字/数字/ハイフン〕**・DB識別子・**会社アバター/アイコン〔アップロード・未設定は頭文字フォールバック〕**・**会社カラー〔`.swatches`〕**＋プロビジョニング注記）。クエストのアイコン/カラー設定（SC-11）と同方式。
- **レイヤー方針**: 運用画面＝業務層のクリーン。

## 4. 表示・入力項目

### 4.1 会社一覧
- **会社名（識別しやすい頭文字タイル＝会社カラー）/ 会社コード（等幅表示）/ DB識別子 / 状態（有効/準備中/停止）/ アカウント数 / グループ数 / 作成日 / 管理導線**。

### 4.2 会社作成モーダル
- **会社名（必須）/ 会社コード（必須・全社一意・`company_code`）/ DB識別子（必須・`.env` 参照キー）/ 会社アバター・アイコン（任意・アップロード／未設定は頭文字＋会社カラー）/ 会社カラー（プリセット `.swatches`）**。作成で管理DBに Company 登録（状態=準備中・`company_code`/`color`/`icon_image_path` 保持）。**会社DBは別途プロビジョニング**（compose）後に有効化。
  - **会社コードのフォーマット**: 半角英大文字/数字/ハイフン・4〜20文字・先頭は英字・大小文字を区別せず大文字正規化・**全社で一意**（重複はエラー）。原則作成時に確定し以後不変。

## 5. アクション・遷移

| トリガー | 条件 | 遷移/結果 |
| --- | --- | --- |
| 会社の行 / 「管理する →」 | — | **SC-92 会社詳細**（`?company=<id>`）へ。以降の設定・アカウント操作はその会社に対して |
| ＋会社を作成 | — | モーダル→管理DBに Company 登録（状態=準備中）。DBプロビジョニング後に有効化 |
| 検索 / 状態フィルタ | — | 一覧を更新 |
| ダッシュボードへ戻る | — | **SC-01** |

## 6. データ・API（想定）

- **GET `/api/sysadmin/companies`**: 会社一覧（名称/会社コード/DB識別子/状態/件数）。検索（会社名/会社コード/DB識別子）・ページング想定。
- **POST `/api/sysadmin/companies`**: 会社作成（Company 登録・状態=準備中）。**`company_code` はサーバーで一意検証（重複=409）＋大文字正規化**。**DBプロビジョニングは運用手順**（MVP: compose 手編集＋`.env`）。
- 権限は**システム管理者のみ**（サーバーで厳格化）。運用操作は監査ログに記録。

## 7. アクセシビリティ / 表示配慮

- 会社は頭文字タイル（会社カラー）＋名称で識別しやすく。状態は色＋テキスト。会社コード・DB識別子は等幅。
- 行全体がクリック可（「管理する →」ボタンも）。破壊的操作なし（作成のみ）。

## 8. 状態・バリエーション

| 状態 | 表示 |
| --- | --- |
| 会社=準備中 | プロビジョニング未完（DB接続確認後に有効化） |
| 会社=停止 | 停止バッジ |
| 該当なし | `.list-empty` |
| 多数のテナント | 検索＋ページング（TBD） |
