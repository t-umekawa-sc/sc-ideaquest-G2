# SC-31 アバター / 着せ替え（画面設計 / 機能詳細）

> 画面遷移図: `doc/画面設計/画面遷移図.md` ／ モック: `doc/画面設計/mocks/SC-31_アバター着せ替え.html` ／ 要件: `doc/要件定義/README.md`（FR-08・第6節「装備 / アイテム」）
> 対象デザイン標準: `doc/画面設計/デザイン標準.md`（**ゲーム層**＝レトロ/ドット・CRTガラス）

- 最終更新: 2026-07-19
- ステータス: ドラフト（機能詳細）

## 1. 概要・目的

- **3Dアバターを表示し、所有済み装備を着せ替える**画面（ゲーム層）。ゲーミフィケーションの「見た目のカスタマイズ」による楽しさ・継続動機の中心。
- アバターは **5スロット（頭 / 顔 / 体 / 手持ち / 背景）**・各スロット1点まで装備。**装備の入手はコイン購入のみ**（ショップ SC-30）で、本画面は**所有済み装備の付け替え**を担う。
- 導線: **共通ヘッダーのアバターメニュー「アバター / 着せ替え」**／ダッシュボード（SC-01）ヒーローの「▶ きせかえ」・ショートカットタイル「アバター」。

## 2. アクセス条件・権限

| 項目 | 内容 |
| --- | --- |
| アクセス可能者 | 認証済みユーザー全員（自分のアバターのみ編集） |
| 装備 | **所有済み（購入済み）装備**のみ着せ替え可。各スロット1点・「外す」で未装備に |
| 購入 | 本画面では行わない。未所有装備は**ショップ（SC-30）へ誘導**。購入・所有はサーバーで検証 |

## 3. レイアウト構成（決定・ゲーム層）

- **共通ヘッダー**（Lv/コイン/SP＋ベル＋アバターメニュー）。本画面ではアバターメニューの「アバター / 着せ替え」が現在地（`aria-current`）。
- **2カラム**（狭い画面〔≤900px〕は1カラム縦積み）:
  - **左＝アバタービューア（`.pixel-panel`＝CRTガラス・sticky）**: 3Dアバターを表示（本番は VRM/R3F で回転可）。氏名・Lv・（モックは装備を絵文字で重ねた見立て）。
  - **右＝ワードローブ**: コイン残高＋**ショップ導線**、スロットごと（頭/顔/体/手持ち/背景）に**所有装備＋「外す」＋未所有（ショップ）**を並べたグリッド。
- **レイヤー方針（1レイヤー1マテリアル）**: アバタービューアは CRTガラス（ゲーム層の顔）。ワードローブのカードは**可読性優先のクリーンカード＋ゲーム要素**（レアリティ色・ピクセル見出し・`.btn-pixel`）。SC-32 と同方針。

## 4. 表示・入力項目

### 4.1 アバタービューア（左・CRTガラス）
- **3Dアバター**（本番＝three-vrm・パーツ差し替え方式でスロット装備メッシュを反映。回転/ズーム）。モックは `mascot-hero.png` ＋装備絵文字の重ねで見立て。
- **氏名・レベル**。装備変更が**即座に反映**される（プレビュー同期）。

### 4.2 ワードローブ（右・スロット別）
- スロットは **5種（頭 / 顔 / 体 / 手持ち / 背景）**、各スロットに「装備中: ◯◯」を表示。
- 各スロットのグリッド先頭に **「✕ 外す」カード**（未装備にする）。続けて装備カード。
- **装備カード**＝サムネイル（本番は装備画像）＋名称＋**レアリティ**（コモン/標準/レア＝`--rarity-*`）。
  - **所有・装備中**: 枠を強調＋「装備中」バッジ。
  - **所有・未装備**: クリックで装備。
  - **未所有**: グレーアウト＋価格（◆N）＋「🔒 ショップで購入」。クリックで**ショップ（SC-30）へ誘導**。
- コイン残高（`◆ N`）と **「🛒 ショップへ」ボタン**を上部に表示。

### 4.3 装備ラインナップ（初期値・調整可）
- **入手はコイン購入のみ**。レアリティ3段階＝コモン10〜30 / 標準50〜150 / レア300〜800コイン（第6節）。
- モック初期セット（例）: 頭＝王冠/シルクハット/キャップ/麦わら帽、顔＝サングラス/メガネ/マスク、体＝アーマー/スーツ/道着/コート、手持ち＝剣/杖/本/大槌、背景＝夕焼け/星空/森/古城。※最終ラインナップは TBD。

## 5. アクション・遷移

| トリガー | 条件 | 遷移/結果 |
| --- | --- | --- |
| 装備カードをクリック | 所有済み | 同スロットの装備を差し替え（1点）。ビューアに即反映。**サーバーは所有を検証** |
| 「✕ 外す」 | — | 当該スロットを未装備に |
| 未所有カード / 「🛒 ショップへ」 | — | **SC-30 ショップ**へ（購入は SC-30 で実施） |
| ダッシュボードへ戻る | — | **SC-01** |

## 6. データ・API（想定）

- **GET `/api/items`**: 装備マスタ（名称/スロット/レアリティ/価格/パーツ参照）＋自分の所有・装備状況（`UserItem`）。
- **PUT `/api/me/avatar`**（or `/api/me/equipment`）: スロットごとの装備を更新。**サーバーで所有（購入済み）を検証**し、各スロット1点に正規化。
- 購入は **SC-30**（`POST /api/items/{id}/purchase`・コイン残高/価格をサーバー検証）。
- **3D**: 装備の VRM パーツをスロットに取り付け・差し替え（three-vrm / R3F）。

## 7. アクセシビリティ / 表示配慮

- 装備カードは `role="button"`・キーボード操作（Enter/Space）対応。装備状態は色だけでなく「装備中」バッジ＋文言で示す。
- レアリティは色＋テキスト併記（`--rarity-*`）。CRTガラス上の明るい文字は on-dark トークン（Lv=`--pixel-level-on-dark` 等）。
- **3D（WebGL）非対応環境**の代替表示は TBD（第10節）。`prefers-reduced-motion` 尊重（回転/演出）。

## 8. 状態・バリエーション

| 状態 | 表示 |
| --- | --- |
| 所有0のスロット | 「外す」＋未所有（ショップ）カードのみ。装備は「なし」 |
| 全スロット未装備 | ベースアバターのみ表示 |
| コイン不足の未所有装備 | ショップで購入不可（SC-30 側で残高検証） |
