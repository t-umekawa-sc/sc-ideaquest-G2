## 1. 概要・目的

- **達成した実績・バッジを一覧表示・収集する**画面（ゲーム層）。活動（投稿/採用/評価/投票/議論/継続/成長/魔法/装備）に応じてバッジを獲得。
- **報酬（決定・2026-07-19 変更）**: 実績を獲得すると**難易度（ティア）に応じてコインを付与**する（**ブロンズ ◆20 / シルバー ◆50 / ゴールド ◆150**・初期値・調整可）。一度きりの恒久ボーナス（グラインド不可）なのでコインの希少性は保つ。※以前の「実績報酬なし」から方針転換。装備の直接付与は行わず、報酬は**コイン**で統一（装備はショップ購入のまま）。
- 導線: **ダッシュボード（SC-01）のショートカットタイル「実績 / バッジ」**／プロフィール等。

## 2. アクセス条件・権限

| 項目 | 内容 |
| --- | --- |
| アクセス可能者 | 認証済みユーザー全員（自分の実績を閲覧） |
| 付与 | **サーバーで活動（`Transaction/Activity` 等）を判定**し `UserAchievement` を作成（恒久・冪等）。同時に**ティア連動コインを付与**（一度きり検証） |
| 報酬 | ティア連動コイン（**ブロンズ ◆20 / シルバー ◆50 / ゴールド ◆150**・初期値）。`Transaction/Activity` に記録 |
| シークレット | 達成するまで名称・条件・アイコン・報酬を伏せる（`？？？`／🔒） |

## 3. レイアウト構成（決定・ゲーム層）

- **共通ヘッダー**（Lv/コイン/SP＋ベル＋アバターメニュー）。
- **収集サマリー（`.pixel-panel`＝CRTガラス）**: 獲得数 `X / Y` ＋**達成率バー（`.xp-bar` 流用）**＋%。
- **注記**: 活動で獲得・**ティア連動コイン報酬（◆20/◆50/◆150）**・シークレットは伏せる旨。
- **絞り込み（一覧標準 `.list-toolbar`）**: カテゴリ／状態（すべて/獲得済み/未獲得）＋件数。
- **バッジグリッド（`.ach-grid`）**: 実績カードのグリッド（`.list-empty` 対応）。
- **レイヤー方針（1レイヤー1マテリアル）**: 収集サマリーは CRTガラス（ゲーム層の顔）。バッジカードは**可読性優先のクリーンカード＋ゲーム要素**（ティア色リング・進捗バー）。SC-30/31/32/41 と同方針。

## 4. 表示・入力項目

### 4.1 収集サマリー
- **獲得数 / 総数**・**達成率（%）バー**・**実績で獲得したコイン合計（◆）**。

### 4.2 バッジカード
- **メダル（アイコン・ティア色リング）＋ティア（ブロンズ/シルバー/ゴールド）＋名称＋説明＋条件＋報酬（◆コイン）**。
  - **獲得済み**: 「✓ 獲得」＋獲得日。カラー表示・「獲得コイン ◆N」。
  - **未獲得（数値系）**: グレー＋進捗バー（`cur / max`）＋「報酬 ◆N」。
  - **未獲得（達成型）**: グレー＋「🔒 未獲得」＋「報酬 ◆N」。
  - **シークレット（未獲得）**: アイコン🔒・名称`？？？`・条件`？？？`・報酬`◆?`（ティア含め内容を伏せる）。獲得後は通常表示＋「シークレット」バッジ。

### 4.3 実績カタログ（初期値・調整可・**ティア連動コイン報酬**）

| カテゴリ | 実績（例） | 条件 | ティア |
| --- | --- | --- | --- |
| アイデア投稿 | 🌱初めの一歩 / 💡アイデアマン / 🏭発明家 | 1件 / 10件 / 50件 投稿 | 銅/銀/金 |
| 採用 | 🎉初採用 / 🏆ヒットメーカー | 1件 / 5件 選定 | 銀/金 |
| 評価 | 👀目利き / 🧙賢者の評 | 10件 / 50件 評価 | 銅/金 |
| 投票 | 🗳️一票の力 / 📊選挙管理人 | 1回 / 50回 投票 | 銅/銀 |
| 議論 | 💬おしゃべり / 🔥議論の達人 | 10件 / 100件 コメント | 銅/金 |
| 継続 | 📅皆勤の芽 / 🗓️皆勤賞 | 7日 / 30日 連続ログイン | 銅/金 |
| 成長 | ⭐レベル5 / 🌟レベル10 | Lv5 / Lv10 到達 | 銀/金 |
| 魔法 | ✨見習い魔導士 / 🪄大魔道士 | 1つ / 全6種 解放 | 銅/金 |
| 装備 | 👕おしゃれさん / 👑コレクター | 1点 / 全装備 購入 | 銅/金 |
| シークレット | 🦉夜更かしフクロウ / 🕵️？？？ | 深夜投稿 ほか（伏せる） | 銀/金 |

- ティアは難易度の段階（ブロンズ/シルバー/ゴールド）で、**獲得時のコイン報酬に連動**（◆20 / ◆50 / ◆150・初期値）。シークレットもティアに準拠。

| ダッシュボードへ戻る | — | **SC-01** |

## 6. データ・API（想定）

- **GET `/api/achievements`**: 実績マスタ（カテゴリ/ティア/アイコン/名称/説明/条件/シークレット）＋自分の獲得状況（`UserAchievement`・獲得日）＋数値系の進捗。
- 付与は**サーバー側で活動イベントから判定**（投稿/選定/評価/投票/コメント/ログイン連続/レベル到達/魔法解放/装備購入 等）。**冪等**（多重付与なし）。獲得時に**ティア連動コインを付与**し `Transaction/Activity` に記録（一度きり検証）。
- **データモデル（案）**: `Achievement`（マスタ・**ティア連動の報酬コインを保持**／`coin_reward`＝ティア既定 or 個別）／`UserAchievement`（User×Achievement・獲得日・進捗）。
- シークレットは未獲得時に名称/条件/報酬をクライアントへ返さない（サーバーで伏せる）。

## 7. アクセシビリティ / 表示配慮

- 状態（獲得/未獲得）を色だけでなく「✓ 獲得」「🔒 未獲得」の文言＋進捗数値で明示。
- ティアは色＋テキスト併記。CRTガラス上の文字は on-dark トークン。
- グレースケール/淡色は装飾（意味は文言で担保）。演出は `prefers-reduced-motion` 尊重。
