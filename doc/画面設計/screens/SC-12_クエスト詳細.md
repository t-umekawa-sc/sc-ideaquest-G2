# SC-12 クエスト詳細（画面設計 / 機能詳細）

> 画面遷移図: `doc/画面設計/画面遷移図.md` ／ モック: `doc/画面設計/mocks/SC-12_クエスト詳細.html` ／ 要件: `doc/要件定義/README.md`（FR-20〜27・権限モデル）
> 対象デザイン標準: `doc/画面設計/デザイン標準.md`（業務層＝クリーン。登録・編集はモーダル標準）

- 最終更新: 2026-07-16
- ステータス: ドラフト（機能詳細）

## 1. 概要・目的

- 1つのクエストの詳細画面。**クエスト情報＋アイデア一覧（内包）＋パーティー/権限**を集約し、アイデアの投稿・議論・評価の起点になる。
- **アイデア一覧は独立画面にせず本画面のタブに内包**（旧 SC-20 廃止）。
- **登録・編集はモーダル**（UI標準）：アイデア追加＝SC-21、クエスト編集/パーティー・権限編集＝SC-11 をモーダルで開く。

## 2. アクセス条件・権限

| 項目 | 内容 |
| --- | --- |
| アクセス可能者 | **当該クエストのパーティーメンバー**（アイデアの公開範囲＝パーティー内のみ）。非メンバーはアクセス不可 |
| 表示 | パーティーなら全員が閲覧可 |
| アクション権限 | アイデア追加＝**アイデア作成権限**／クエスト編集・パーティー/権限編集＝**所有者 or クエスト管理権限**／評価＝評価者権限（SC-25）。権限の無いアクションボタンは非表示 |
| 管理者閲覧 | 管理機能の範囲での閲覧は別（本画面はパーティー向け） |

## 3. レイアウト構成（決定）

- **戻る導線**（→SC-10 クエスト一覧）
- **上部2カラム**: 左＝**クエストヘッダー（カード）**（カテゴリー・ステータス・件名・目的/テーマ・メタ＋アクション）、右＝**クエスト内 週間ランキング**（ゲーム層 `.pixel-panel .rank-panel`）。
- **タブ切替**（決定）: `[アイデア] [パーティー] [概要]`。既定＝**アイデア**タブ。
- **アイデア一覧＝テーブル（行リスト）**（決定）。

> レイアウトは「**上部2カラム（左クエスト情報＋右クエスト内週間ランキング）＋タブ（アイデア/パーティー/概要）／アイデア一覧はテーブル**」で確定（2026-07-16）。狭幅（〜860px）では2カラムを縦積み。

## 4. 表示項目

### 4.1 クエストヘッダー
| 項目 | 種別 | 補足 |
| --- | --- | --- |
| クエストアイコン | `.quest-icon lg` | カスタム画像 or **未設定時は件名頭文字＋所有者アバターのフォールバック**。設定はクエスト編集モーダル（SC-11） |
| カテゴリー | `.badge badge-muted` | |
| ステータス | `.badge` | 募集中/進行中/評価中/選定/完了（下書きは本画面に出ない＝作成者のみ） |
| 件名 | 見出し | |
| 目的・テーマ | テキスト | |
| メタ | テキスト | 締切（近接は警告色）/ 👥人数 / 💡アイデア数 / 所有者 / グループ |
| ＋アイデアを追加 | ボタン(primary) | アイデア作成権限。**SC-21 モーダル**を開く |
| クエスト編集 | ボタン(outline) | 所有者/管理権限のみ表示。**SC-11 モーダル**を開く（**クエストアイコンの設定/解除**を含む） |

### 4.1b クエスト内 週間ランキング（右カラム・ゲーム層）
| 項目 | 種別 | 補足 |
| --- | --- | --- |
| 見出し / 期間 | テキスト | 「★ クエスト内ランキング ★」＋「このクエストの活動で獲得（今週）」 |
| 順位（1〜3位） | メダル | 🥇🥈🥉。自分が3位内なら行ハイライト＋（あなた） |
| アバター / 氏名 | `.avatar sm` ＋テキスト | パーティーメンバー |
| スコア | 数値 | **このクエストの活動で得た XP＋コインの合算**（内訳 EXP/◆ 併記） |

> **スコア定義**: `スコア = 今週このクエストの活動で得た XP ＋ 得たコイン`。全社の週間ランキング（SC-01）と同じ合算式だが、**集計対象を当該クエストの活動に限定**（Activity の `quest_id` で絞り込み）。週次リセット。共通コンポーネント `.rank-panel`（shared.css）を使用。

### 4.2 タブ: アイデア（既定・テーブル）
- **一覧ツールバー（標準：検索＋フィルター＋並び替え、`.list-toolbar`）**:
  - **検索**: 件名・投稿者のキーワード。
  - **フィルター**: 状態（すべて/未投票/投票済/自分の投稿）・評価（すべて/評価待ち/評価済）。将来は投稿者などを追加。
  - **並び替え**: 投票が多い順 / 新着順 / 評価が高い順 / コメントが多い順。
  - 件数表示（`.list-count`）・「絞り込みをクリア」・0件時の空表示（`.list-empty`）。
- テーブル列: **件名 / 投稿者（アバター＋氏名）/ 賛成・反対（▲n ▼n）/ 💬コメント数 / 評価（n/5 or 評価待ち）/ あなた（未投票・投票済・自分の投稿）**。
- **行クリックで SC-22 アイデア詳細**（本番は Intercept Route でモーダル差し込み、URL直/リロードはフルページ）。
- 投票の集計表示は会社の匿名化設定に従う（既定＝集計数）。

### 4.3 タブ: パーティー
- メンバー一覧（アバター＋氏名＋**権限バッジ**：所有者/クエスト管理/評価者/投票/作成/コメント）。
- **パーティー・権限を編集**ボタン（所有者/管理のみ）＝ SC-11 モーダル（メンバー追加/削除・権限付与/剥奪）。候補は同一クエストグループ所属者に限定。
- 補足: 新規参加メンバーの既定権限＝投票＋作成＋コメント。パーティーから外すと権限は失うが既存の入力は表示継続。

### 4.4 タブ: 概要
- 定義リスト: ステータス / カテゴリー / 目的・テーマ（全文）/ 期限日 / クエストグループ / 所有者 / 作成日 / アイデア数。

## 5. アクション・遷移

| トリガー | 条件 | 遷移/結果 |
| --- | --- | --- |
| 戻る | — | SC-10 クエスト一覧 |
| タブ切替 | — | アイデア/パーティー/概要を切替（同一URL・本番は `?tab=` 等） |
| アイデア行クリック | パーティー | **SC-22 アイデア詳細**（モーダル差し込み） |
| ＋アイデアを追加 | アイデア作成権限 | **SC-21 アイデア作成モーダル**。投稿で XP・コイン付与、一覧に反映 |
| クエスト編集 | 所有者/管理 | **SC-11 クエスト編集モーダル** |
| パーティー・権限を編集 | 所有者/管理 | **SC-11 モーダル**（メンバー/権限） |
| 並び替え | — | 一覧の順序変更 |

## 6. 状態・空表示

| 状況 | 表示 |
| --- | --- |
| アイデアが0件 | テーブルの代わりに空状態：「まだアイデアがありません」＋（権限があれば）＋アイデアを追加への導線 |
| 締切超過 | ステータス＝評価中/完了などに応じた表示。投稿/投票の可否はサーバー判定 |
| アクション権限なし | 対応ボタンを非表示（クリック不可） |
| ローディング | ヘッダー/テーブルはスケルトン（実装） |

## 7. API 入出力（案・API設計フェーズで確定）

- **GET `/api/quests/{id}`**: クエスト情報（カテゴリー/テーマ/期限/ステータス/所有者/グループ/自分の権限）。
- **GET `/api/quests/{id}/ideas?q=&state=&eval=&sort=&page=`**: アイデア一覧（件名/投稿者/賛成数/反対数/コメント数/評価集計/自分の投票状態）。**検索・フィルター（状態/評価）・並び替え・ページング**に対応（一覧標準）。※匿名化設定に従い集計のみ返す場合あり。
- **GET `/api/quests/{id}/members`**: パーティー＋各メンバーの権限。
- **GET `/api/quests/{id}/ranking/weekly?limit=3`**: クエスト内週間ランキング（当該クエストの活動〔Activity.quest_id〕から今週の XP＋コインを合算）。TOP3＋自分の順位。
- **POST `/api/quests/{id}/ideas`**（SC-21）: アイデア作成（下書き/公開）。
- **PATCH `/api/quests/{id}`**（SC-11）: クエスト編集。**メンバー/権限**は `/members`・`/permissions` 系で更新。
- すべて当該会社DB＋パーティー/権限をサーバーで検証。クロステナント禁止。

## 8. 非機能・アクセシビリティ

- タブは `role=tablist/tab/tabpanel`・`aria-selected`。キーボード操作対応（実装で ←→ 移動）。
- テーブル行リンクはキーボードでも到達可能にする（実装：各行に代表リンク or `tabindex`＋Enter）。
- 投票集計・評価は色のみに依存しない（数値＋テキスト）。
- 背景画像（`.app-bg`）は全認証画面共通のため本画面でも反映。
