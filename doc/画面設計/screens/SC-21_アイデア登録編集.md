# SC-21 アイデア登録・編集（画面設計 / 機能詳細）

> 画面遷移図: `doc/画面設計/画面遷移図.md` ／ モック: `doc/画面設計/mocks/SC-21_アイデア登録編集.html` ／ 要件: `doc/要件定義/README.md`（FR-20 系アイデア・FR-26 添付・アイデアの入力項目）
> 対象デザイン標準: `doc/画面設計/デザイン標準.md`（業務層＝クリーン。登録・編集はモーダル標準）

- 最終更新: 2026-07-18
- ステータス: ドラフト（機能詳細）

## 1. 概要・目的

- 参加中クエストに**アイデアを新規登録／編集**する画面。**必須は 件名・アイデア本文・価値**の 3 項目、他は任意（投稿ハードルを下げる）。
- UI 標準に合わせ**独立画面ではなくモーダル**（Parallel Routes ＋ Intercept Routes・URL 直/リロードはフルページ）。
- 開く導線: **SC-12 クエスト詳細「アイデア追加／編集」／ SC-01 下書きパネル（続きを編集）**。
- 投稿で**パーティーに公開・アイデアごとのチャットグループを自動作成・投稿 XP を付与**。**下書き保存**なら本人のみ表示（付与なし）。

## 2. アクセス条件・権限

| 項目 | 内容 |
| --- | --- |
| 登録 | 当該クエストの**アイデア作成権限**を持つパーティーメンバー（新規参加者は既定で保有） |
| 編集 | 原則**投稿者本人**（管理者の扱いは §9 TBD） |
| 公開範囲 | 投稿は**そのクエストのパーティー内のみ**（会社/グループ全体には非公開） |

## 3. レイアウト構成（決定）

- **モーダル**（`.modal` 標準）。パネル内に上から:
  1. **ドラッグ用ヘッダー**（⠿グリップ＋「アイデアを登録／編集」）
  2. **投稿先クエストの文脈**（クエストアイコン＋クエスト名＋カテゴリ）＋権限注記
  3. **必須 3 項目**（件名 → 価値 → アイデア本文）
  4. **任意項目（折りたたみ）**＝タイムリミット・利害関係者・備考（既定は閉。投稿ハードルを下げる）
  5. **関連資料 添付**（ドロップゾーン＋添付チップ・任意・複数可）
  6. フッター（投稿する・**下書き保存**・キャンセル）
- 業務層のクリーン基調。**ヘッダーをドラッグで移動可**。**必須3項目が揃うまで「投稿する」は無効**。

## 4. 表示・入力項目（アイデアの入力項目）

| 項目 | 必須 | 入力 | 備考 |
| --- | --- | --- | --- |
| 件名 | **必須** | text | アイデアのタイトル |
| 価値 | **必須** | textarea | もたらす価値・メリット（**評価の要**なので必須） |
| アイデア本文 | **必須** | textarea | 内容の説明 |
| タイムリミット | 任意 | date | 実施/検討の想定期限（折りたたみ内） |
| 利害関係者 | 任意 | **複数選択コンボボックス**（`.multiselect data-free`） | 関係する人・部署を**複数選択可・自由入力可**（折りたたみ内） |
| 備考 / 特記事項 | 任意 | textarea | 補足（折りたたみ内） |
| 関連資料 | 任意 | file（複数） | 添付。実体は MinIO・物理名ハッシュ化・DB はパス＋元名（FR-26） |

- **必須3項目**（件名・本文・価値）を上部に固定、**任意は `<details>` に格納**。
- 添付は**クリック選択＋ドラッグ＆ドロップ**、種別アイコン＋ファイル名＋サイズ＋削除。上限サイズ/許可形式は TBD。

## 5. アクション・遷移

| トリガー | 条件 | 遷移/結果 |
| --- | --- | --- |
| 必須3項目を入力 | — | 「投稿する」を有効化 |
| 任意項目を開く | — | 折りたたみ展開 |
| ファイルを添付/削除 | — | 添付チップを追加/除去（MinIO 実体は投稿時にアップロード） |
| 投稿する | 必須充足・作成権限 | 公開・**チャットグループ自動作成**・**投稿 XP 付与**、閉じる（SC-12/SC-22 へ反映） |
| **下書き保存** | — | **本人のみ表示**の下書き（付与なし）。SC-01 下書きパネルから再開可 |
| ヘッダーをドラッグ | — | モーダル移動 |
| キャンセル / ✕ / バックドロップ | — | 破棄して閉じる |

## 6. 状態・バリエーション

| 状態 | 表示 |
| --- | --- |
| 新規登録 | 空フォーム・投稿ボタン無効 |
| 編集 | 既存値/添付を読み込み「更新」 |
| 下書きの続き | 保存済み下書きを読み込み継続編集（公開まで本人のみ） |
| 必須未充足 | 投稿ボタン無効（どの必須が空かは `*` で明示） |
| 作成権限なし | この画面に到達しない（SC-12 に追加ボタンを出さない） |
| 締切/クローズ後 | 可否はステータス＋サーバー判定（§9 TBD） |

## 7. データ・API（想定）

- **POST `/api/quests/{qid}/ideas`** `{subject, value, body, time_limit?, stakeholders?, note?, attachments[], status: draft|published}`: 新規。`published` でチャットグループ作成＋投稿 XP。
- **PUT `/api/ideas/{id}`**: 編集（本人・サーバー権限検証）。
- **POST `/api/attachments`**（multipart）: MinIO へアップロード→パス＋元名を返す（物理名ハッシュ化）。ダウンロードは権限検証＋署名付き URL。
- 必須（件名・本文・価値）は**サーバーでも検証**。`draft` は本人のみ・付与なし。

## 8. アクセシビリティ / 表示配慮

- 必須は `*`（`.req`）＋ボタン無効化で明示。任意項目の折りたたみは `<details>`（キーボード可）。
- ドロップゾーンは `role="button"`＋キーボード（Enter/Space）で選択可。
- モーダル標準（フォーカストラップ・`aria-modal`・`Esc`/バックドロップで閉じる・`prefers-reduced-motion`）。
