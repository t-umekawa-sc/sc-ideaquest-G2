# SC-25 評価画面（画面設計 / 機能詳細）

> 画面遷移図: `doc/画面設計/画面遷移図.md` ／ モック: `doc/画面設計/mocks/SC-25_評価画面.html` ／ 要件: `doc/要件定義/README.md`（評価・第6節）
> 対象デザイン標準: `doc/画面設計/デザイン標準.md`（業務層＝クリーン。登録・編集はモーダル標準）

- 最終更新: 2026-07-18
- ステータス: ドラフト（機能詳細）

## 1. 概要・目的

- 評価者が 1 つのアイデアを **5 観点 × 各 5 点満点** で採点し、**観点別コメント**を付ける画面。
- **SC-22 アイデア詳細の「評価する / 編集」から開く**。UI 標準に合わせ **モーダル**（Intercept Route・URL 直/リロードはフルページ）。
- 1 アイデアに **複数の評価者**が評価しうる。各評価者は**自分の評価を入力/編集**する。

## 2. アクセス条件・権限

| 項目 | 内容 |
| --- | --- |
| アクセス可能者 | **評価者権限**を持つパーティーメンバーのみ（作成者は既定で評価者、＋付与された参加メンバー） |
| 新規/編集 | 自分の評価が未登録なら新規、あれば内容を読み込み「更新」 |
| 締切後 | 投稿/評価の可否はステータス＋サーバー判定（TBD） |

## 3. レイアウト構成（決定）

- **モーダル**（`.modal` 標準）。パネル内に上から:
  1. **ドラッグ用ヘッダー**（⠿グリップ＋タイトル「アイデアを評価」）＋権限注記
  2. **対象アイデアの文脈**（クエスト名・カテゴリ・件名・「アイデア詳細を見る（SC-22）」）
  3. **アイデアを確認（折りたたみ）**＝価値/本文/添付を `<details>` で埋め込み（既定オープン・移動しなくても内容確認可）
  4. **5 観点の採点行**（観点名＋説明＋★スター〔1〜5〕＋数値＋観点別コメント textarea）
  5. **集計プレビュー**（本評価の平均・採点数・コイン付与の説明）
  6. **公開範囲**（パーティー全員〔既定〕/ 限定）
  7. フッター（評価を確定 / **下書き保存** / キャンセル）
- 業務層のクリーン基調。★の塗りはコイン色（アンバー）。
- **モーダルはヘッダーをつかんで移動可能**（背後の SC-22 を確認する用途）。画面内に収める。バックドロップは薄め。
- **「アイデアを確認」の折りたたみ埋め込み**と**ドラッグ移動**の両方でアイデア内容を参照できる（リロード/フルページや狭幅では折りたたみが確実）。

## 4. 表示・入力項目

### 4.1 5 観点の採点
| 観点 | 説明 | 配点 |
| --- | --- | --- |
| 新規性 | 目新しさ・独自性 | ★1〜5 |
| 影響度 | 実現時のインパクト | ★1〜5 |
| 実現度 | 実現可能性の高さ | ★1〜5 |
| 適合性 | クエストのテーマ/目的への合致 | ★1〜5 |
| **コスト** | **低コストほど高得点（★5＝非常に低コスト）** を明示（ラベル/ヘルプで誤解防止） | ★1〜5 |

- 各観点に **★スター（クリックで点数・ホバーでプレビュー・キーボード可）** ＋ **観点別コメント（任意）**。
- **5 観点は均等平均**（重みなし）。

### 4.2 集計プレビュー
- **本評価の平均**（採点済み観点で算出・全 5 観点で確定）＋採点数（n/5）。
- コインは**全評価者の平均**から `round(平均×10)`（最大 50）で**投稿者へ付与**（本画面は本評価の平均のみ表示）。

### 4.3 公開範囲（評価結果の閲覧範囲・決定）
| 選択 | 内容 |
| --- | --- |
| 🔓 パーティー全員（**既定**） | 参加メンバー全員がスコア・観点別コメントを閲覧可 |
| 🔒 限定 | 投稿者＋評価者＋所有者/クエスト管理権限者のみ。範囲外への見せ方（集計のみ/非表示）は TBD |

- データ: `Evaluation.visibility`（`party`/`limited`）。閲覧可否はサーバー側で最終制御。

## 5. アクション・遷移

| トリガー | 条件 | 遷移/結果 |
| --- | --- | --- |
| ★クリック | 評価者 | 該当観点の点数を設定・平均を再計算 |
| 観点別コメント入力 | 評価者 | 保持 |
| 公開範囲の選択 | 評価者 | `party`/`limited` を選択 |
| 評価を確定/更新 | 評価者・**全 5 観点採点済み** | 評価を確定保存、パーティー/他評価者へ反映、**評価実施の XP＋投稿者コインを付与**、閉じる |
| **下書き保存** | 評価者 | **本人のみ表示の下書き**として一時保存（全観点そろわなくても可・XP/コインは付与しない）。閉じても再開可 |
| ヘッダーをドラッグ | — | モーダルを移動（背後の SC-22 を確認） |
| アイデアを確認（折りたたみ） | — | 価値/本文/添付をモーダル内で開閉表示 |
| キャンセル / ✕ / バックドロップ | — | 破棄して閉じる（SC-22 に戻る） |
| アイデア詳細を見る | — | **SC-22 アイデア詳細** |

## 6. 状態・バリエーション

| 状態 | 表示 |
| --- | --- |
| 新規評価 | 全観点未採点（–）から入力 |
| 下書きあり | 保存済み下書きを読み込み継続編集。確定するまで本人のみ表示 |
| 既存評価の編集 | 確定済みスコア/コメント/公開範囲を読み込み「更新」 |
| 未採点あり | 確定時に警告（全観点必須） |
| 締切/クローズ後 | 可否はサーバー判定（不可時は入力無効化＋理由）→ TBD |
| 評価者権限なし | 本画面に到達しない（SC-22 に評価ボタンを出さない） |

## 7. データ・API（想定）

- **GET `/api/ideas/{id}/evaluation/me`**: 自分の既存評価/下書き（各観点スコア/コメント・公開範囲・`status`）。無ければ空。
- **PUT `/api/ideas/{id}/evaluation`** `{scores{...}, comments{...}, visibility, status: draft|submitted}`: 登録/更新。**`submitted` は全観点必須をサーバー検証**し、評価実施 XP＋**投稿者のコインを全評価者平均から再計算**（確定タイミング〔締切一括/都度〕は実装で整理）。**`draft` は本人のみ・付与なし**。
- 評価結果の閲覧（SC-22）は `visibility` に従いサーバーで制御（範囲外は集計のみ/非表示・TBD）。

## 8. アクセシビリティ / 表示配慮

- ★は色のみに依存しない（数値併記・`aria-label`「n点」・スターグループに `role`）。
- コスト観点は「低コストほど高得点」を明示（色＋アイコン＋テキスト）。
- モーダル標準（フォーカストラップ・`aria-modal`・`Esc`/バックドロップで閉じる・`prefers-reduced-motion`）。
