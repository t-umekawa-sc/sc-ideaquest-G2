# SC-10 クエスト一覧（画面設計 / 機能詳細）

> 画面遷移図: `doc/画面設計/画面遷移図.md` ／ モック: `doc/画面設計/mocks/SC-10_クエスト一覧.html` ／ 要件: `doc/要件定義/README.md`（FR-15 参照制限・一覧標準）
> 対象デザイン標準: `doc/画面設計/デザイン標準.md`（一覧標準・クエストアイコン・色アクセント・モーダル標準）

- 最終更新: 2026-07-18
- ステータス: ドラフト（機能詳細）

## 1. 概要・目的

- 自分が参加中のクエストを一覧する画面。**「所属クエストグループ内で作られ、かつ自分がパーティー参加中」**のクエストのみ表示（FR-15）。
- 一覧から **SC-12 クエスト詳細**へ、**「＋クエストを作成」から SC-11 作成モーダル**へ。
- **一覧標準**（検索＋フィルター＋並び替え＋件数/空表示）を適用。**表示は カード / リスト（テーブル）を切替**。

## 2. アクセス条件・権限

| 項目 | 内容 |
| --- | --- |
| アクセス | 認証済みユーザー |
| 表示対象 | **所属グループ内 × 自分がパーティー参加中**のクエスト（会社/グループ全体には非公開） |
| 作成 | 認証済みなら誰でもクエスト作成可（作成者＝所有者） |
| 下書き | 下書き中のクエストは**本人のみ**（本一覧ではなくダッシュボード SC-01 のパネルに表示） |

## 3. レイアウト構成（決定）

- **ページ見出し「クエスト一覧」＋「＋クエストを作成」**（右上・SC-11 モーダル）。
- **一覧標準ツールバー**（`.list-toolbar`）: 左＝検索＋状態＋グループの絞り込み（絞り込み中は「クリア」）、右＝件数＋並び替え＋**表示切替（カード/リスト）**。
- **カードビュー**（既定）: `.card.card-accent.quest-card` を `.quest-grid` に並べる。クエストアイコン＋**クエスト色アクセント**（帯＋アイコンタイル `--accent`）。
- **リストビュー**: `.table`（`.table-wrap`）。件名/カテゴリ/ステータス/締切/👥/💡/あなた。行クリックで SC-12。
- 空表示は `.list-empty`。背景画像（`.app-bg`）反映。

## 4. 表示項目

### 4.1 カード / 行 共通のクエスト情報
| 項目 | 補足 |
| --- | --- |
| クエストアイコン | カスタム画像 or 未設定時は**件名頭文字＋所有者アバター**。色は `--accent`（クエスト色） |
| 件名 | クエスト名 |
| カテゴリ | `.badge badge-muted` |
| ステータス | 募集中/進行中/評価中/選定/完了（完了=muted・選定=success） |
| 締切 | 近接は警告色（`.deadline.soon`） |
| 👥 パーティー人数 / 💡 アイデア数 | 参加人数・アイデア件数 |
| あなた | 自分の状態（未投稿=danger / 投稿済み=success 等） |
| （カードのみ）目的・テーマ | 検索対象（表示は将来／現状メタは締切等） |

### 4.2 ツールバー
| 種別 | 内容 |
| --- | --- |
| 検索 | 件名・テーマ・カテゴリの部分一致 |
| 状態フィルター | すべて/募集中/進行中/評価中/選定/完了 |
| グループフィルター | すべて/所属クエストグループ別（複数所属を想定） |
| 並び替え | 新着順 / 締切が近い順 / アイデアが多い順 / 参加人数が多い順 |
| 件数 / クリア | `.list-count`／絞り込み中は「クリア」表示 |
| 表示切替 | **カード / リスト**（`.viewtoggle`・セグメント） |

## 5. アクション・遷移

| トリガー | 条件 | 遷移/結果 |
| --- | --- | --- |
| カード/行クリック | パーティー | **SC-12 クエスト詳細** |
| ＋クエストを作成 | 認証済み | **SC-11 作成モーダル**（作成後は SC-12 へ） |
| 検索/フィルター/並び替え | — | 一覧を更新（件数連動・0件は空表示） |
| 表示切替 | — | カード ⇄ リストを切替 |
| 戻る | — | ダッシュボード（SC-01） |

## 6. 状態・バリエーション

| 状態 | 表示 |
| --- | --- |
| 0件（未参加/絞り込み） | 空表示（`.list-empty`） |
| 締切超過 | 締切表示（ステータス連動）。完了は muted |
| 未投稿のクエスト | あなた＝未投稿（danger）で気づける |
| 多数 | クライアント側で処理しきれない量はサーバー側フィルター/ソート/**ページング**へ（TBD） |

## 7. データ・API（想定）

- **GET `/api/quests?scope=joined&q=&status=&group=&sort=&page=`**: 参加中クエスト一覧（アイコン/件名/カテゴリ/ステータス/締切/人数/アイデア数/自分の状態/グループ）。**参照制限（所属グループ×参加）はサーバーで強制**。
- 件数に応じ**クライアント側 or サーバー側**でフィルター/ソート/ページング（一覧標準）。
- 作成は SC-11（`POST /api/quests`）。

## 8. アクセシビリティ / 表示配慮

- ステータス/自分の状態は色のみに依存しない（バッジ＝色＋テキスト）。
- 表示切替は `aria-pressed`。行リンクはキーボード到達（本番は行内リンク/ロール付与を検討）。
- クエスト色アクセントは装飾（意味はテキストで担保）。
