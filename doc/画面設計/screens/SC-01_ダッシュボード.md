# SC-01 ダッシュボード（画面設計 / 機能詳細）

> 画面遷移図: `doc/画面設計/画面遷移図.md` ／ モック: `doc/画面設計/mocks/SC-01_ダッシュボード.html` ／ 要件: `doc/要件定義/README.md`
> 対象デザイン標準: `doc/画面設計/デザイン標準.md`（業務層＝クリーン、ゲーム層＝レトロ・ドット絵）

- 最終更新: 2026-07-16
- ステータス: ドラフト（機能詳細）

## 1. 概要・目的

- ログイン後の **ホーム画面**。自分の状況（レベル/XP・コイン・アバター）と、**参加中クエストへの直行**・通知・各機能への導線を1画面に集約する。
- **二層構成**: 上部の **2カラム（左＝ステータスヒーロー / 右＝週間ランキングTOP3）＝ゲーム層**（`.pixel-panel`）で見せ、以降のクエスト/通知/管理導線は **業務層**（クリーンなカード）。
- **ログインボーナス演出**: ログイン直後、**画面下部トーストに RPG 風メッセージを自動送り**で表示し、数秒でフェードアウト（`.pixel-dialog` を土台）。XP付与は SC-00 のログイン処理側で確定済み（1日1回・日次上限対象）。ここでは**結果の演出のみ**。

## 2. アクセス条件・権限

| 項目 | 内容 |
| --- | --- |
| アクセス可能者 | 認証済みユーザー全員。未認証は SC-00 へリダイレクト |
| ロール | 一般 / クエストグループ管理者 / システム管理者。**ロールにより管理導線（SC-90/SC-91）の表示可否が変わる** |
| テナント | ログインで確定した所属会社DBのデータのみ表示（クロステナント不可） |

## 3. レイアウト構成（決定）

上から順に:

1. **アプリヘッダー**（共通 `.app-header`）: ロゴ / Lv ピル / コインピル / 通知ベル（未読バッジ）/ アバターメニュー。
2. **あいさつ**: 「おかえりなさい、〇〇 さん」＋一言。
3. **上部2カラム（ゲーム層）**: 左＝**ヒーロー**（アバター / レベル / XPバー / コイン / SHOP・きせかえ導線）、右＝**週間ランキングTOP3**。
4. **下書き**: 下書き中のクエスト/アイデアのカードパネル。**最上段**。クリックで作成/編集モーダル（SC-11/SC-21）を開いて続きを編集。**下書きが無ければ非表示**。
5. **未投票のアイデア**: 参加クエストで自分が未投票のアイデアのパネル（**下書きの下**）。**カード内で賛成/反対クイック投票**、パネルクリックで**拡大→投票モーダル**。**未投票が無ければ非表示**。
6. **参加中クエスト**: カードグリッド。カード全体クリックで **SC-12 直行**。「すべて見る」→ SC-10。
7. **フォロー中のアイデア**: カードパネルのグリッド。**パネルクリックで拡大アニメーション → アイデア詳細（SC-22）をモーダル表示**。位置は**参加中クエストの直下**。
8. **下段2カラム**: 左＝最近の通知（→SC-02）、右＝ショートカットタイル。
9. **管理導線（ロール条件付き）**: SC-90 / SC-91。

> レイアウト方針は「**上部2カラム（左ヒーロー＋右 週間ランキング）＋下書き＋未投票アイデア＋参加中クエスト＋フォロー中アイデア＋下段**」で確定（2026-07-16）。狭幅（〜860px）では各2カラムを1カラムに畳む。

### 4.3 週間ランキング TOP3（ゲーム層・→SC-41）
| 項目 | 種別 | 補足 |
| --- | --- | --- |
| 見出し / 期間 | テキスト | 「★ 週間ランキング ★」＋「今週の獲得EXP＋コイン」。集計期間は**今週（週次リセット）** |
| 順位（1〜3位） | メダル | 🥇🥈🥉。同点時のタイブレークは TBD（10節） |
| アバター / 氏名 | `.avatar sm` ＋テキスト | **自分が3位内なら行をハイライト＋（あなた）** |
| ウィークリースコア | 数値 | **今週の獲得XP＋獲得コインの合算**（内訳 EXP/◆ を併記） |
| すべて見る | リンク | SC-41 ランキング（全順位・自分の順位） |

> **週間ランキングのスコア定義（初期値・調整可）**: `ウィークリースコア = 今週の獲得XP + 今週の獲得コイン` の**単純合算**で順位付け。XP・コインは「その週に新たに獲得した分」を対象（累計ではない）。**週次でリセット**。※ XP と コインの重み・週の起点（月曜/日曜・タイムゾーン）・同点処理は 10節 TBD。
> ※ ランキングは会社（テナント）内で集計。**投票匿名化の設定とは独立**（ランキングは本人のXP/コイン実績であり投票者開示ではない）。

### 4.4 下書き（作成者本人のみ・最上段）
| 項目 | 種別 | 補足 |
| --- | --- | --- |
| 下書きバッジ | `.badge badge-draft` | 「下書き」（警告色系） |
| 種別バッジ | `.badge badge-muted` | クエスト / アイデア |
| タイトル | テキスト | 入力途中の件名（未入力なら「無題」） |
| メタ | テキスト | クエストの場合＝カテゴリー/期限、アイデアの場合＝所属クエスト。最終更新日時 |
| 続きを書く | CTA | クリックで **作成/編集モーダル（SC-11/SC-21）** を開く |

- **下書き＝作成者本人のみ表示**（公開/投稿までパーティー・他メンバーに非公開）。**XP・コインは公開/投稿時に付与**（下書きでは付与しない）。
- 下書きが1件も無ければ**セクションごと非表示**（ダッシュボードを圧迫しない）。
- **パネルクリックで拡大アニメーション → 作成/編集モーダル（SC-11/SC-21）が開く**（フォローパネル→詳細と同じ共有要素アニメーション＝framer-motion `layoutId`）。モーダル内は種別に応じ**クエスト編集フォーム / アイデア編集フォーム**を表示し、フッターは「公開する/投稿する」＋「下書き保存して閉じる」。モックは vanilla JS の FLIP＋簡易フォームで実演。

### 4.5 未投票のアイデア（投票ショートカット）
| 項目 | 種別 | 補足 |
| --- | --- | --- |
| 件名 | テキスト | カードタイトル |
| 未投票バッジ | `.badge badge-muted` | 「未投票」 |
| 所属クエスト / カテゴリー | テキスト | どのクエストのアイデアか |
| 価値（要約） | テキスト | 2行程度で省略 |
| クイック投票 | ボタン（賛成/反対） | **カード内で即投票**（モーダルを開かない）。賛成=グリーン/反対=レッド |

- 対象＝**参加クエストのアイデアで、自分がまだ投票していないもの**（FR-23）。投票すると **XP付与（初期値5・日次上限）**。1人1票・締切まで変更可・自分のアイデアにも投票可。
- **クイック投票**: カードの賛成/反対で即確定 → カードは投票済み表示（「✓ 賛成/反対 で投票しました」）→ 未投票リストから外れる。
- **パネルクリック → 拡大モーダル**: パネルが拡大して **投票モーダル**（アイデア要約：クエスト/カテゴリー/価値/本文＋現在の集計＋大きな賛成/反対）。投票=**SC-22内のアクションと同一**（本番はダッシュボードから投票モーダルを差し込み or SC-22 の投票UIを共用）。
  - モックは vanilla JS の FLIP＋簡易モーダルで実演（下書き/フォローと同じ `flipOpen/flipClose`）。
- **匿名化**: 表示上の集計は会社設定に従う（既定=匿名＝集計数のみ）。投票アクション自体は常に可能。
- 未投票が0件なら**セクションごと非表示**。

### 4.6 参加中クエストカード（クリックで SC-12）
| 項目 | 種別 | 補足 |
| --- | --- | --- |
| クエストアイコン | `.quest-icon sm` | カスタム画像 or 未設定時は件名頭文字＋所有者アバター |
| 件名 | テキスト | カードタイトル |
| 状態バッジ | `.badge` | 募集中 / 評価中 / 選定 等（クエストのフェーズ） |
| カテゴリー | `.badge badge-muted` | |
| 締切 | テキスト | 残日数。**近接（例3日以内）は警告色**（`.deadline.soon`） |
| パーティー人数 / アイデア数 | テキスト | 👥 / 💡 |
| 自分の状態バッジ | `.badge` | 未投稿 / 投稿済み / 評価待ち 等（自分に紐づくアクション喚起） |

> 表示対象＝「所属クエストグループ内で作られ、かつ自分がパーティー参加中」のクエスト（要件どおり）。件数が多い場合は**直近・締切近い順に数件**＋「すべて見る」で SC-10。

### 4.7 フォロー中のアイデア（パネル→拡大モーダル）
| 項目 | 種別 | 補足 |
| --- | --- | --- |
| 件名 | テキスト | パネルタイトル |
| 所属クエスト | テキスト | どのクエストのアイデアか |
| 価値（要約） | テキスト | 2行程度で省略 |
| 投票 / コメント数 | テキスト | ▲賛成 / ▼反対 / 💬 |
| フォロー★ | ボタン | ON=ゴールド★。**クリックでフォロー解除トグル**（パネルは開かない） |

- **フォロー＝ウォッチ**。フォロー中アイデアに動き（新規コメント・評価・選定 等）があると通知（FR-28）。表示源は会社DBの **Follow**（User×Idea）。
- **パネルクリック → 拡大モーダル**: パネルがそのまま拡大して **アイデア詳細（SC-22）をモーダル表示**。
  - **実装（決定）**: 本番は **Intercept Route で SC-22 を差し込み**（URL は `/idea/xx`、リロード/直アクセスはフルページ）。**パネル→モーダルの拡大は framer-motion の `layoutId`**（共有要素アニメーション）。モックは vanilla JS の FLIP 簡易版で見せる。
  - モーダル内: 所属クエスト / カテゴリー / フォロー状態 / 価値 / 本文 / 投票 / 「アイデア詳細を開く（SC-22 フルページ）」。`Esc`・バックドロップ・×で閉じ、閉じるとパネルへ縮小して戻る。
- 表示対象が多い場合は**直近動きのあった順に数件**（「すべて見る」導線は将来）。フォロー0件時は空状態（7節）。

### 4.8 最近の通知（業務層・→SC-02）
- 直近数件（メンション / 自アイデアへのコメント / 選定・XP獲得 等）。未読は左に赤ドット。各行クリックで対象（SC-22/SC-24 等）へ。「すべての通知」→ SC-02。

### 4.9 ショートカットタイル
- クエスト一覧(SC-10) / ショップ(SC-30) / アバター(SC-31) / 実績(SC-40) / ランキング(SC-41) / 通知(SC-02)。

### 4.10 管理導線（ロール条件付き）
- クエストグループ管理者 → **SC-90** を表示。システム管理者 → **SC-91** を表示。一般ユーザーには非表示。

### 4.11 背景画像アップロード（右上・全認証画面に反映）
- あいさつ行の**右上**に「🖼️ 背景画像」ボタン（設定済みなら「リセット」も表示）。
- クリックで**画像ファイルを選択**し、**コンテンツ領域の背景**に設定。設定は**ユーザー個人設定**として保存され、**その利用者の全認証画面に反映**（本番はユーザー設定APIに保存し認証済みレイアウトへ共通適用・実体は MinIO）。
- 可読性のため背景の上に**薄い明色スクリム**を重ねる（`.app-bg`・デザイン標準参照）。
- モックは **localStorage** に保存し、別画面へ移動しても残る＝全画面反映を簡易再現。

## 6. ログインボーナス演出（確定）

- **表示方法＝画面下部トースト・自動送り**（2026-07-16 決定）。`.pixel-dialog` を土台にした下部固定トースト。
- 文字は **RPG風タイプライター**（1文字ずつ）。送り終えて数秒後に自動フェードアウト。**クリック不要・操作を邪魔しない**。
- `prefers-reduced-motion` 時は**即時全文表示**して数秒で消える。
- 表示条件はログイン処理の付与結果に従う（**1日1回・日次上限は SC-00/バックエンドで判定済み**。ダッシュボードは付与結果を受け取って演出するのみ）。
- レベルアップを伴う場合の演出（例：トーストを `.levelup` にする / 続けてレベルアップ表示）は**別途詰める TBD**（10節）。

## 8. API 入出力（案・API設計フェーズで確定）

> 実際のパス/認証方式はAPI設計で確定。ダッシュボードは複数ソースの集約表示。

- **GET `/api/me/dashboard`**（集約）
  - out: `{ user:{...}, drafts:[{id, kind:"quest"|"idea", title, subtitle, updated_at}], unvoted_ideas:[{id, title, quest, category, value, body, agree, disagree, deadline}], quests:[{id, title, category, phase, deadline, party_count, idea_count, my_status}], followed_ideas:[{id, title, quest, category, value, agree, disagree, comments, updated_at}], notifications:[...], unread_count, roles:[...], weekly_ranking:[...], login_bonus?:{...} }`
  - ※ 実装では `/api/me`・`/api/drafts`・`/api/ideas?unvoted=1&limit=n`・`/api/quests?participating=1&limit=n`・`/api/ideas?followed=1&limit=n`・`/api/notifications?limit=n`・`/api/ranking/weekly?limit=3` に分割する可能性あり（API設計で決定）。
- **投票**: `POST /api/ideas/{id}/vote { dir: "agree"|"disagree" }`（変更は同APIで上書き・締切内のみ）。会社DBの Vote を upsert し、投票者に XP 付与（初期値5・日次上限）。表示集計は匿名化設定に従う。投票済みは `unvoted_ideas` から外れる。
- **下書き**: `drafts` は **作成者本人の下書き（Quest.status=下書き / Idea.status=下書き）**のみ。公開/投稿は各作成モーダルの API（例 `POST /api/quests`・`PATCH .../publish`、`POST /api/ideas`・`.../publish`）で status を遷移し、その時点で XP・コインを付与。
- **フォロー操作**: `POST /api/ideas/{id}/follow` / `DELETE /api/ideas/{id}/follow`（会社DBの Follow を作成/削除）。可否は「当該アイデアが自分の参加クエストか」を検証。動きの通知は Notification で配信。
- **モーダル詳細**: フォローパネルから開く SC-22 は **Intercept Route（`/idea/{id}`）**。データ取得は SC-22 の詳細APIを共用（モーダルとフルページで同一）。
- **週間ランキング**: サーバーで**今週の獲得XP＋獲得コインを合算**して会社内で順位付け（`score = weekly_xp + weekly_coins`）。ダッシュボードは TOP3＋自分の順位を受け取る。全順位は SC-41。集計はトランザクション（Activity）ログの週次集計で算出。
- **login_bonus** はログイン処理（SC-00 の `/api/auth/*`）で確定した付与結果を、初回ダッシュボード取得時に一度だけ返す想定（多重表示を防ぐ）。
- 表示は所属会社DB＋管理DB（ロール）を参照。**クロステナント禁止**。

## 9. 非機能・アクセシビリティ

- カード全体リンクはキーボードフォーカス可（`<a>`）。フォーカスリング・:hover/:active の押下フィードバックを付与。
- 通知の未読ドットは色のみに依存しない（アイコン＋テキストで種別が分かる）。
- ゲーム層（ヒーロー・トースト）のアニメーションは `prefers-reduced-motion` で抑制。
- 数値（Lv/XP/コイン）はサーバー値を正とし、クライアントはキャッシュ表示（差異時は再取得）。
