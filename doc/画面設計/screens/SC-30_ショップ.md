# SC-30 ショップ（画面設計 / 機能詳細）

> 画面遷移図: `doc/画面設計/画面遷移図.md` ／ モック: `doc/画面設計/mocks/SC-30_ショップ.html` ／ 要件: `doc/要件定義/README.md`（FR-07・第6節「装備 / アイテム」「コイン付与ルール」）
> 対象デザイン標準: `doc/画面設計/デザイン標準.md`（**ゲーム層**＝レトロ/ドット・CRTガラス）

- 最終更新: 2026-07-19
- ステータス: ドラフト（機能詳細）

## 1. 概要・目的

- **コインで装備アイテムを購入**する画面（ゲーム層）。装備の**入手経路はコイン購入のみ**（レベル解放・実績報酬なし）で、本画面がその唯一の入手口。
- 購入した装備は **きせかえ（SC-31）** でスロットに着せ替える。SC-30＝買う／SC-31＝着る、の役割分担。
- コインは**アイデアの評価点に連動**して貯まる希少通貨（行動では付与しない）。装備の希少性を担保。
- 導線: **ダッシュボード（SC-01）ヒーローの「▶ SHOP へ」・ショートカットタイル**／**きせかえ（SC-31）の未所有カード・「🛒 ショップへ」**。

## 2. アクセス条件・権限

| 項目 | 内容 |
| --- | --- |
| アクセス可能者 | 認証済みユーザー全員 |
| 購入 | **自分のコイン残高 ≥ 価格**のとき購入可。**サーバーで残高・価格・所有をサーバー検証**（第7節・不正入手防止） |
| 入手方法 | **コイン購入のみ**（レベル解放・実績報酬なし）。購入は恒久（`UserItem` を作成） |

## 3. レイアウト構成（決定・ゲーム層）

- **共通ヘッダー**（Lv/コイン/SP＋ベル＋アバターメニュー）。ヘッダーのコインは購入で即更新。
- **コイン残高ヒーロー（`.pixel-panel`＝CRTガラス）**: 残高を大きく表示＋「評価点連動で貯まる」「装備はコイン購入のみ／着せ替えは SC-31」の要点＋**「▶ きせかえへ」**（`.btn-pixel`）。
- **レアリティ帯の説明**: コモン ◆10〜30 / 標準 ◆50〜150 / レア ◆300〜800（色付き）。
- **絞り込み・並び替え（一覧標準 `.list-toolbar`）**: スロット / レアリティ絞り込み＋「買えるものだけ」「未所有だけ」トグル＋並び替え＋件数。
- **装備グリッド（`.shop-grid`）**: 装備カードのグリッド（`.list-empty` 対応）。
- **レイヤー方針（1レイヤー1マテリアル）**: コイン残高ヒーローは CRTガラス（ゲーム層の顔）。装備カードは**可読性優先のクリーンカード＋ゲーム要素**（レアリティ色リング・`.btn-pixel`）。SC-31 / SC-32 と同方針。

## 4. 表示・入力項目

### 4.1 コイン残高ヒーロー
- **コイン残高**（大表示）／評価点連動の注記／**きせかえ（SC-31）への導線**。

### 4.2 絞り込み・並び替え（一覧標準）
- **スロット絞り込み**（すべて/頭/顔/体/手持ち/背景）。
- **レアリティ絞り込み**（すべて/コモン/標準/レア）。
- **「買えるものだけ」**（価格 ≤ 残高 の未所有＋所有済み）／**「未所有だけ」**トグル。
- **並び替え**（レアリティ順〔既定〕/価格が安い順/価格が高い順/スロット順）。件数表示。

### 4.3 装備カード
- サムネイル（本番は装備画像。**レアリティ色のリング**）＋スロット・レアリティ＋名称＋**価格（◆N）**。
  - **未所有・購入可**: 「購入する」（`.btn-pixel`）。
  - **未所有・コイン不足**: ボタン無効＋「コイン不足（あと ◆n）」・価格を警告色。
  - **所有済み**: 「✓ 所有済み」＋「▶ きせかえで装備」（SC-31 へ）。

### 4.4 ラインナップ（初期値・調整可・SC-31 と同一セット）

| スロット | 装備（例） |
| --- | --- |
| 頭 | 👑王冠(レア ◆600) / 🎩シルクハット(標準 ◆120) / 🧢キャップ(コモン ◆20) / 👒麦わら帽(コモン ◆20) |
| 顔 | 🕶️サングラス(標準 ◆90) / 👓メガネ(コモン ◆15) / 😷マスク(コモン ◆15) |
| 体 | 🛡️アーマー(レア ◆700) / 👔スーツ(標準 ◆150) / 🥋道着(コモン ◆30) / 🧥ロングコート(標準 ◆120) |
| 手持ち | ⚔️剣(レア ◆500) / 🪄魔法の杖(標準 ◆120) / 📖本(コモン ◆25) / 🔨大槌(標準 ◆90) |
| 背景 | 🌅夕焼けの海(標準 ◆100) / 🌌星空(レア ◆400) / 🌲森(コモン ◆25) / 🏰古城(レア ◆500) |

- 価格帯: **コモン ◆10〜30 / 標準 ◆50〜150 / レア ◆300〜800**（第6節・初期値）。

## 5. アクション・遷移

| トリガー | 条件 | 遷移/結果 |
| --- | --- | --- |
| 購入する | 未所有 かつ 残高 ≥ 価格 | 確認 → **サーバーで残高/価格/所有を検証** → コインを減算・`UserItem` を作成（恒久）。カードを「所有済み」へ・残高（本文/ヘッダー）を更新 |
| きせかえで装備 / ▶ きせかえへ | — | **SC-31 アバター/着せ替え**へ |
| 絞り込み・並び替え | — | クライアント（本番は必要ならサーバー）で一覧を更新 |
| ダッシュボードへ戻る | — | **SC-01** |

## 6. データ・API（想定）

- **GET `/api/items`**: 装備マスタ（名称/スロット/レアリティ/価格/パーツ参照）＋自分の所有状況（`UserItem`）。※SC-31 と共用。
- **POST `/api/items/{id}/purchase`**: 購入。**サーバーでコイン残高・価格・重複所有を検証**し、`coin_balance` を減算・`UserItem`（所有）を作成・`Transaction/Activity`（消費）に記録。**冪等**（二重購入防止）。
- 着せ替え（装備反映）は **SC-31**（`PUT /api/me/avatar`）。装備の実体は VRM パーツ（three-vrm）。

## 7. アクセシビリティ / 表示配慮

- レアリティは色（リング）＋テキスト併記。価格・購入可否は文言でも示す（コイン不足はボタン無効＋残額）。
- CRTガラス上の明るい文字は on-dark トークン。コイン色（`--pixel-coin`）は XP/レベル/SP と区別。
- 購入直後の演出（`just-bought`）は `prefers-reduced-motion` を尊重。購入は確認ダイアログで誤操作防止。

## 8. 状態・バリエーション

| 状態 | 表示 |
| --- | --- |
| コイン0 / 不足 | 買えない装備はボタン無効＋「コイン不足（あと ◆n）」。「買えるものだけ」で購入可能品に絞れる |
| 全所有 | 全カード「✓ 所有済み」＋きせかえ導線。未所有0件 |
| 絞り込み該当なし | `.list-empty`（条件見直しを促す） |
