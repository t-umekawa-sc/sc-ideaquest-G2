# SC-92 会社詳細 / 設定（システム管理）（画面設計 / 機能詳細）

> 画面遷移図: `doc/画面設計/画面遷移図.md` ／ モック: `doc/画面設計/mocks/SC-92_会社詳細.html` ／ 要件: `doc/要件定義/README.md`（FR-16・第2/8節・データモデル Company/Account／会社DB quest_group_members）
> 対象デザイン標準: `doc/画面設計/デザイン標準.md`（**業務層**＝クリーン。トグルは `.switch`）
> 関連: **SC-91 システム管理（会社一覧）** から遷移（`?company=<id>`）

- 最終更新: 2026-07-19
- ステータス: ドラフト（機能詳細）

## 1. 概要・目的

- **システム管理者（運営）**が、**選択した 1 社（テナント）**の**設定**（投票匿名化／管理者開示／MFA）と**アカウント・クエストグループ所属のフル操作**を行う画面。
- SC-91（会社一覧）で会社を選ぶと本画面へ。**画面上部に「メンテナンス中の会社」を示す文脈バナーを常時（sticky）表示**し、どの会社を操作しているかを明確にする（誤操作防止）。
- 会社一覧（SC-91）と分離することで、テナント単位の管理が完結し、視点が明確になる。

## 2. アクセス条件・権限

| 項目 | 内容 |
| --- | --- |
| アクセス可能者 | **システム管理者（運営）のみ** |
| 対象 | URL の `?company=<id>` で特定した**1社**。以降の設定/アカウント操作はすべてこの会社に適用 |
| データ | 設定＝`Company`（会社コード/フラグ）・`Account`（管理DB）／**所属＝会社DB `quest_group_members`**（QG所属は会社DBが正・§8-①） |

## 3. レイアウト構成（決定・業務層）

- **共通ヘッダー**（運営バッジ＋ベル＋アバターメニュー）。
- **パンくず**: システム管理 › 会社一覧 › ◯◯株式会社。
- **文脈バナー（`.ctx`・sticky）**: **会社アイコン〔`.quest-icon`＝画像 or 頭文字タイル・会社カラー〕＋「メンテナンス中の会社」ラベル＋会社名（大）＋会社コード（等幅）＋状態バッジ＋DB識別子＋件数＋「会社一覧へ戻る」**。会社カラーで左帯を色付けし識別性を上げる。**共通ヘッダーの直下（`top: --header-h`）で固定**し、スクロールしても表示。**z-index はヘッダーより低く**（ヘッダーと重ならず、アバターメニュー等のヘッダー内UIが常に手前）。
- **会社プロフィールカード**: **会社アバター/アイコン（アップロード・未設定は頭文字フォールバック）＋会社カラー（`.swatches`）**。変更は**バナーのアイコン・カラー（左帯）に即反映**。
- **会社設定カード**: **`.switch` トグル3種**＋DB識別子＋プロビジョニング注記。
- **アカウント & 所属（この会社）**: 一覧標準（検索＋状態フィルタ）＋テーブル＋「＋アカウント発行」（会社は固定表示）。
- **モーダル**: アカウント発行/編集（対象会社を固定表示・所属グループはこの会社の候補）。

## 4. 表示・入力項目

### 4.1 文脈バナー（メンテ中の会社）
- **会社アイコン**（画像 or 頭文字タイル・会社カラー）／会社名（大・ピクセル見出し）／**会社コード（等幅・読み取り専用）**／状態／DB識別子／アカウント数・グループ数／会社一覧へ戻る。会社アイコン＋カラーで識別。会社コードは作成時確定・ここでは変更不可。

### 4.1b 会社プロフィール（アバター・カラー）
- **会社アバター/アイコン**（アップロード／未設定は頭文字＋会社カラー・`Company.icon_image_path`）＋**会社カラー**（プリセット `.swatches`・`Company.color`）。クエスト（SC-11）と同方式。変更はバナー・一覧に反映。

### 4.2 会社設定（`.switch`・Company フラグ）

| 設定 | フラグ | 既定 | 補足 |
| --- | --- | --- | --- |
| 投票の匿名化 | `vote_anonymized` | ON | ON=集計数のみ（匿名）／OFF=賛成/反対のアバターを表示（記名） |
| 匿名時に所有者/管理者へ投票者を隠す | `hide_voters_from_managers` | ON | ON=管理者にも非開示／OFF=所有者・クエスト管理者のみ確認可。**匿名化 OFF（記名）時は無効** |
| MFA を必須にする | `mfa_required` | ON | ON=メールOTP（信頼済み端末はスキップ）／OFF=ID＋パスワードのみ |

- **DB接続識別子**（`.env` 参照キー）＋**プロビジョニング注記**（compose 手編集・物理分離）。

### 4.3 アカウント & 所属（この会社）
- **氏名 / ログインID / メールアドレス / システムロール（一般 / クエストグループ管理者 / システム管理者）/ 所属クエストグループ（複数可）/ 状態 / 操作**。※ログインIDとメールは別項目。
- 発行/編集モーダル＝**対象会社（固定表示）**・氏名・**ログインID（識別子）**・**メールアドレス（OTP/初回設定リンク送付先）**・ロール・**所属グループ（`.multiselect`・複数選択・候補のみ〔自由入力なし〕＝この会社のクエストグループから選択）**。

## 5. アクション・遷移

| トリガー | 条件 | 遷移/結果 |
| --- | --- | --- |
| 会社アバター/カラー変更 | — | `Company.icon_image_path`/`color` を更新。**バナー・一覧の表示に即反映**（画像は MinIO 保管） |
| 設定トグル（3種） | — | 当該会社の `Company` フラグを更新（サーバー保存）。匿名化OFFで「管理者へ隠す」は自動無効 |
| ＋アカウント発行 | — | モーダル→**この会社に** Account 作成（管理DB）→会社DBに User ミラー生成＋所属（会社DB `quest_group_members`）割当。初回ログインでパスワード設定 |
| 所属・編集 | — | ロール変更・**所属グループの割当変更**（フル操作） |
| 会社一覧へ戻る / パンくず | — | **SC-91** |
| 検索 / 状態フィルタ | — | この会社のアカウント一覧を更新 |

## 6. データ・API（想定）

- **GET `/api/sysadmin/companies/{id}`**: 会社の詳細＋フラグ＋所属グループ候補。
- **PATCH `/api/sysadmin/companies/{id}`**: `color` / `icon_image_path`（会社アバター・カラー）を更新。アイコン画像は MinIO に保管（ハッシュ名）。
- **PATCH `/api/sysadmin/companies/{id}/settings`**: `vote_anonymized` / `hide_voters_from_managers` / `mfa_required` を更新。
- **GET/POST/PATCH `/api/sysadmin/companies/{id}/accounts`**: この会社のアカウント発行・編集（ロール・所属）。所属割当は**会社DB `quest_group_members`**（QG所属は会社DBが正）。
- 権限は**システム管理者のみ**。操作は監査ログ（`SystemAdmin 操作ログ`）に記録。

## 7. アクセシビリティ / 表示配慮

- **文脈バナーで対象会社を常時明示**（sticky・会社カラー）。誤って別会社を操作しないための最重要要素。
- トグルは `.switch`（focus-visible・ON/OFF 状態語）。依存関係（匿名化OFF→管理者開示は無効）を UI と文言で明示。
- 状態・ロールは色＋テキスト。DB識別子は等幅。破壊的操作は確認。

## 8. 状態・バリエーション

| 状態 | 表示 |
| --- | --- |
| 会社=準備中/停止 | バナーに状態バッジ。設定は可（停止中の扱いは TBD） |
| 匿名化 OFF（記名） | 「管理者へ隠す」トグルは無効・注記表示 |
| アカウント該当なし | `.list-empty` |
| `?company` 不正 | 一覧（SC-91）へ誘導（本モックは先頭会社にフォールバック） |
