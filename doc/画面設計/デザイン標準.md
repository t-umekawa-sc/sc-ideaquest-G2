# デザイン標準（ideaquest）

> 全画面モック（`doc/画面設計/mocks/SC-xx_*.html`）と将来の Next.js 実装が参照する **見た目の土台**。
> 実体のトークンは `doc/画面設計/mocks/shared.css`（CSS変数）に定義。目視確認は `mocks/style-guide.html` をブラウザで開く。

- 最終更新: 2026-07-15
- ステータス: 確定（基本方針）／個別コンポーネントは画面ごとに追記

## 0. コンセプト（決定事項）

- **トーン = バランス型**。基本はクリーンな業務SaaS（企業の社内利用に耐える落ち着き・信頼感）。
  そのうえで **ゲーミフィケーション部分だけ「昔のドット絵ゲーム（レトロ・ピクセルアート）」** をコンセプトにアクセント化する **二層構成**。
- **カラー基調 = ブルー系**（信頼・定番）。
- **モック実装方式 = 共通CSS＋デザイントークン（CSS変数）**。依存なしでブラウザ直開き可、Next.js へトークン単位で移植しやすい。

### 二層の使い分け（重要）
| 層 | 使う画面/領域 | 見た目 |
| --- | --- | --- |
| **業務層（クリーン）** | ログイン・クエスト・アイデア・評価・チャット・フォーム全般・管理画面 | ブルー基調、角丸、ソフトな影、Noto Sans JP |
| **ゲーム層（レトロ）** | ダッシュボードのステータス・XP/レベル・コイン・ショップ・アバター/着せ替え・実績/バッジ・ランキング | ドット絵風、角丸なし、ハードシャドウ、DotGothic16、`.pixel*` クラス |

> ゲーム層は「ご褒美・楽しさ」を担う**アクセント**。業務フローの本体（入力・議論・評価）はクリーン層で迷いなく使えることを優先する。

## 1. カラートークン

`shared.css` の `:root` に定義。主要なもの:

- **ブランド**: `--color-primary #2563EB`（hover `#1D4ED8` / active `#1E40AF` / soft `#EFF6FF`）
- **ニュートラル**: `--color-bg #F8FAFC` / `--color-surface #FFFFFF` / `--color-border #E2E8F0` / `--color-text #0F172A` / `--color-text-muted #64748B`
- **意味色**: success `#16A34A` / warning `#D97706` / danger `#DC2626` / info `#0EA5E9`
- **投票**: 賛成 `--color-agree #16A34A`（グリーン）/ 反対 `--color-disagree #DC2626`（レッド）
- **レアリティ（装備）**: コモン `#94A3B8` / 標準 `#3B82F6` / レア `#A855F7`
- **ゲーム**: XP `#FBBF24` / コイン `#F59E0B` / レベル `#8B5CF6` / cyan `#22D3EE` / magenta `#EC4899` / パネル背景 `#1E293B`

## 2. タイポグラフィ

- **本文フォント（業務層）**: `Noto Sans JP`（フォールバック system-ui）。
- **ゲームフォント（ゲーム層のみ）**: `DotGothic16`（日本語対応のドットフォント）。`.pixel` / `.pixel-panel` / `.btn-pixel` などに適用。
- **サイズスケール**: xs12 / sm14 / base16 / lg18 / xl20 / 2xl24 / 3xl30。行間 1.6。
- モックはCDN（Google Fonts）で読み込む。**実装では `next/font` に置換**する。

## 3. スペーシング / 角丸 / 影

- **余白**: 4px基準（`--space-1`=4 … `--space-16`=64）。
- **角丸**: 標準 8px（`--radius`）/ カード 12px（`--radius-lg`）/ 丸 `--radius-full`。**ゲーム層は 0**（`--radius-pixel`）。
- **影**: 業務層はソフト（`--shadow-sm/…-lg`）。ゲーム層は**ハードシャドウ** `--shadow-pixel: 4px 4px 0`。ボタン押下（`:active`）の“押し込み”は**業務層＝内側の影 `inset`**／ゲーム層＝ハードシャドウ縮小で表現。

## 4. コンポーネント（共通CSSに用意済み）

- **ボタン**: `.btn` / `.btn-primary` / `.btn-outline` / `.btn-danger` / `.btn-sm`、ゲーム用 `.btn-pixel`。
  - **状態**: hover（塗り/枠を一段濃く。`.btn-danger` も hover あり）／ **押下 `:active`**（業務層＝**1px 沈み込み＋内側の影 `inset`＋塗りを一段濃く**のクリーンな挙動。バリアントごとに `active` 色を定義：primary=`--color-primary-active`／danger=`--color-danger-active`／outline=`--color-border`／標準=`--color-border-strong`）。`prefers-reduced-motion` 時は沈みを止め色変化のみ。
  - **ゲーム層 `.btn-pixel`** はハードシャドウで**ガコッと沈む**別トーン（業務層のクリーンな沈みとは意図的に差別化）。
- **カード**: `.card` / `.card-title`。
- **バッジ**: `.badge` / `.badge-success` / `.badge-danger` / `.badge-muted`。投票表示 `.vote-agree` / `.vote-disagree`。
- **レアリティ枠**: `.rarity-common/standard/rare`。
- **ユーザアバター**: `.avatar`（`.sm`/`.lg` でサイズ可変）。`.avatar__img`（画像・なければ `.placeholder` でイニシャル）＋ `.avatar__level`（**下部中央にレベル表示**）。`data-name` 属性＋`tabindex="0"` で**ホバー/フォーカス時にユーザ名ツールチップ**。複数並べるときは `.avatar-group`（少し重ねる）。投票の記名モード・パーティー一覧・チャット等で使用。
- **フォーム**: `.field` + `.input` / `.textarea` / `.select`、必須マーク `.req`、補足 `.hint`。
- **コンボボックス（選択）**: 既定は**ネイティブ `.select`**（a11y・モバイルに強い。本体は指ポインター）。ただし**候補(option)はOS描画のため cursor 制御不可**。候補までホバーの指ポインター/ハイライト/装飾を効かせたい場合は**カスタム `.combobox`**（`<button>＋role=listbox`）を使う。挙動は `mocks/shared.js`（`[data-combobox]` を初期化・↑↓/Enter/Escキー対応）。候補は自前要素なので `cursor:pointer` が確実に効く。
- **チェックボックス**: `.checkbox`（`<label class="checkbox"><input type="checkbox"> 文言</label>`）。**モダン設計の原則に準拠**:
  - ネイティブ `<input>` を `appearance:none` で装飾（キーボード/スクリーンリーダー対応を維持）。
  - **ラベルごとクリック可・ラベル/ボックス両方が指ポインター**（無効時 `not-allowed`）。
  - 状態: hover（枠がブランド色）/ **focus-visible（リング）** / checked（ブランド色＋白チェック）/ **indeterminate**（横棒・JSで設定）/ disabled。
  - ボックスは**ラベル1行目に揃え**、複数行でも崩れない。モーションは ~120ms・`prefers-reduced-motion` で無効。
  - **使い分け**: フォーム内の選択・同意・複数選択はチェックボックス。**即時反映のON/OFF設定（管理画面のMFA有効・投票匿名化 等）はトグルスイッチ**を用いる（`.switch` は今後追加予定）。
- **アプリシェル**: `.app-header` + `.brand`。
- **モーダルダイアログ（標準・`.modal` / `.modal__backdrop` / `.modal__panel`）**: **登録・編集は原則モーダルで行う**（個別の全画面遷移にしない）。詳細表示（アイデア詳細 等）も一覧/ダッシュボードから開く際はモーダル。業務層のクリーンなトーン（白面・角丸 `--radius-lg`・`--shadow-lg`・暗いバックドロップ）。`Esc`/バックドロップクリック/×で閉じる、フォーカストラップ、`aria-modal`。
  - **実装（決定・本番 Next.js）**: **Parallel Routes（`@modal` スロット）＋ Intercept Routes（`(.)`）** で URL を持つモーダルにする（共有・戻る/進むで開閉・**リロード/直アクセスはフルページ**で同一 URL 対応）。
  - **開くアニメーション**: 一覧カードやダッシュボードのパネルから開く場合は、**パネルがそのまま拡大してモーダルになる共有要素アニメーション（framer-motion `layoutId`）**。単独で開く場合はフェード＋わずかなスケールアップ。`prefers-reduced-motion` で動きを抑制。
  - **モックでの見せ方**: vanilla JS の簡易版（FLIP でカード→モーダルへ拡大）。本番の作り込みは画面作成フェーズ。
- **ゲーム層**: `.pixel-panel`（暗い背景パネル）/ `.pixel-stat`（コイン/レベルのピル）/ `.xp-bar`（XPバー）/ `.btn-pixel`（**ホバーでレトロRPGの選択カーソル ▶ が点滅＋1ドット持ち上がる**。`prefers-reduced-motion` で点滅停止）。
- **メッセージウィンドウ / 文字送り**: `.pixel-dialog`（RPG風メッセージ枠。`.pixel-dialog__title` 見出し・`.pixel-dialog__cont ▼` 続き表示・`.levelup` で見出し色変更）。**XP獲得・レベルアップの通知演出**に使う。文字は**RPG風に1文字ずつ表示（タイプライター）**＝実装ではJSで1文字ずつ流し込み（`.pixel-caret` が末尾キャレット）。`prefers-reduced-motion` 時は即時全文表示。見本は `style-guide.html` の 7 節で実演。
- **カーソル**: マウスポインターはOS標準の指差しポインター（`cursor: pointer`）を使用する（カスタムのドット絵カーソルは不採用）。
- **マスコット**: `.mascot`（`assets/mascot-hero.png`＝青マントの剣士ドット絵）。**カードにホバーするとカード右上に出現し、別カードへホバーが移るとカード間を飛んで移動**する。ページに1体だけ配置し、位置をホバー中カードの右上へ更新（`left/top` トランジションで飛ぶ＋待機時 `mascot-bob` でふわふわ）。`prefers-reduced-motion` で動きを抑制。
  - **モックは vanilla JS の簡易版**。**本番(Next.js)のカード間なめらか移動は共有要素アニメーション（framer-motion の `layoutId` 等）で実装＝画面作成フェーズに持ち越し**。
- **レイアウトユーティリティ**: `.container` / `.stack` / `.row` / `.row-center` / `.between` / `.grid` / `.muted` など。

## 5. レイアウト規約

- 中央寄せコンテナ幅 `--container: 1120px`、ヘッダー高 `--header-h: 56px`。
- ブレークポイント: 768px 以下でモバイル（`.row` を縦積み、コンテナ余白を縮小）。
- 各画面モックの基本骨格:
  ```html
  <link rel="stylesheet" href="shared.css">
  <header class="app-header"><div class="container between">…</div></header>
  <main class="container"> … </main>
  ```

## 6. モック運用ルール

- 1画面 = 1ファイル `doc/画面設計/mocks/SC-xx_<名前>.html`。必ず `shared.css` を参照（トークンを画面内に直書きしない）。
- 画面固有の微調整のみ各HTMLの `<style>` に書く。**再利用する見た目は `shared.css` へ昇格**。
- モックは静的プロトタイプ（確認用）。ロジックは持たない。実装（Next.js）とは別管理。
- 新しい色・部品が必要になったら、まず本ドキュメント＋`shared.css` にトークン/クラスを追加してから使う。
