## 1. プロジェクト概要

| 項目 | 内容 |
| --- | --- |
| プロジェクト名 | ideaquest（アイデアクエスト） |
| 一言でいうと | ゲーム感覚で楽しくアイデアを出し・管理できる WEB アプリ |
| 解決したい課題 | アイデア出し・管理は続けにくく単調になりがち。継続の動機付けが弱い |
| ターゲットユーザー | 企業（社内でアイデアを集めたい組織）。実際に操作するのは企業の従業員と管理者 |
| 提供価値 | ゲーミフィケーション（経験値・レベル・コイン・3Dアバター装備）で、社内のアイデア創出を楽しく活性化・継続できる |

## 2. 背景・目的

企業では社内から広くアイデアを集めたいというニーズがあるが、アイデア出しは地味で継続しづらく、従業員の参加も鈍りがち。
ideaquest は「ゲーム感覚で楽しく操作できる」ことをテーマに、行動（アイデア投稿・ログインなど）に対して
経験値やコインといった報酬を与え、レベルアップやアバターの装飾を通じて、社内のアイデア創出を活性化・習慣化させることを目的とする。
企業単位で導入し、その企業の従業員がユーザーとして参加する形態を想定する。

## 3. スコープ

### やること（In Scope）

- アイデアの登録・管理（一覧・編集・削除など基本 CRUD）
- ゲーミフィケーション要素
  - 経験値（XP）の獲得とレベルアップ
  - コインの獲得
  - ショップでの装備購入
  - 3D アバターへの装備反映
- ユーザー登録・ログイン（ログインでも XP 獲得）
- 企業（テナント）単位の管理
  - 企業ごとにユーザー・アイデアを分離（マルチテナント）
  - 管理者による従業員ユーザーの管理
  - 管理者向けのアイデア閲覧・集計

### やらないこと（Out of Scope）

- （TBD: 例）多人数リアルタイム共同編集、SNS 連携 等 → 要判断

## 4. ユーザー / ペルソナ

| ペルソナ | 概要 | 主なニーズ |
| --- | --- | --- |
| 一般従業員 | アイデアを投稿する主役ユーザー | 楽しく手軽にアイデアを出したい・成長やアバターを楽しみたい |
| 管理者（企業担当者） | 自社のユーザー・アイデアを管理する | アイデアを集約・活用したい、利用状況を把握したい、メンバーを管理したい |
| （TBD）運営 / システム管理 | サービス提供側の全体管理 | 企業（テナント）の管理、アイテムの管理 等 |

## 5. 主要ユースケース / ユーザーストーリー

- [ ] ユーザーとして、思いついたアイデアを素早く登録したい。登録すると経験値がもらえて楽しいから
- [ ] ユーザーとして、毎日ログインして経験値を貯めたい。継続の動機になるから
- [ ] ユーザーとして、貯めたコインで装備を買い、3D アバターを着せ替えたい。自分らしくカスタマイズしたいから
- [ ] ユーザーとして、レベルが上がる実感を得たい。成長を感じたいから

## 6. 機能要件

| ID | 機能 | 概要 | 優先度 |
| --- | --- | --- | --- |
| FR-01 | アイデア登録 | アイデアを新規登録する。登録時に XP・コインを付与 | Must |
| FR-02 | アイデア管理 | 一覧表示・編集・削除 | Must |
| FR-03 | ユーザー認証 | 登録・ログイン・ログアウト | Must |
| FR-04 | 経験値・レベル | 行動に応じて XP を付与し、閾値でレベルアップ | Must |
| FR-05 | ログインボーナス | ログイン（例: 1日1回）で XP を付与 | Should |
| FR-06 | コイン | 行動に応じてコインを付与、残高を管理 | Must |
| FR-07 | ショップ | コインで装備アイテムを購入 | Must |
| FR-08 | 3D アバター | アバター表示、購入した装備の着せ替え・反映 | Must |
| FR-09 | 実績 / バッジ | （案）達成に応じた実績付与 | Could |
| FR-10 | ランキング | （案）社内での XP やレベルのランキング表示 | Could |
| FR-11 | 企業（テナント）管理 | 企業単位でユーザー・データを分離して管理 | Must |
| FR-12 | 権限・ロール | 一般従業員／管理者などの役割ごとに操作を制御 | Must |
| FR-13 | 管理者機能 | 従業員ユーザーの管理、投稿アイデアの閲覧・集計 | Should |
| FR-14 | SSO 連携 | （案）企業の認証基盤（SAML/OIDC 等）との連携 | Could |

## 9. データモデル

（主要エンティティの案。詳細は別ファイル `データモデル.md` に分割予定）

- **Company / Tenant**: 企業（テナント）。ユーザーやアイデアの所属先
- **User**: ユーザー。所属企業、ロール（一般／管理者）、XP、レベル、コイン残高を保持
- **Idea**: アイデア。所有ユーザー、所属企業、タイトル、本文、作成日時
- **Item / Equipment**: 装備アイテム。名称、種別、価格、3Dモデル参照
- **UserItem**: ユーザーが所有・装備中のアイテム（多対多）
- **Transaction / Activity**: XP・コインの付与・消費履歴（不正防止・監査用、案）
