## 1. プロジェクト概要

| 項目 | 内容 |
| --- | --- |
| プロジェクト名 | ideaquest（アイデアクエスト） |
| 一言でいうと | ゲーム感覚で楽しくアイデアを出し・管理できる WEB アプリ |
| 中心概念 | アイデア出しを「**クエスト**」という単位でまとめる。テーマ・期限・参加メンバー（パーティー）を決めて取り組み、メンバーがアイデアを出し合い、投票・議論・評価する |
| 解決したい課題 | アイデア出し・管理は続けにくく単調になりがち。継続の動機付けが弱い |
| ターゲットユーザー | 企業（社内でアイデアを集めたい組織）。実際に操作するのは企業の従業員と管理者 |
| 提供価値 | ゲーミフィケーション（経験値・レベル・コイン・3Dアバター装備）で、社内のアイデア創出を楽しく活性化・継続できる |

## 2. 背景・目的

企業では社内から広くアイデアを集めたいというニーズがあるが、アイデア出しは地味で継続しづらく、従業員の参加も鈍りがち。
ideaquest は「ゲーム感覚で楽しく操作できる」ことをテーマに、行動（アイデア投稿・ログインなど）に対して
経験値やコインといった報酬を与え、レベルアップやアバターの装飾を通じて、社内のアイデア創出を活性化・習慣化させることを目的とする。
企業単位で導入し、その企業の従業員がユーザーとして参加する形態を想定する。

## 3. スコープ

### やること（In Scope）

- **クエスト**（アイデア出しのテーマ単位）の作成・管理
  - カテゴリー・目的/テーマ・期限日・参加メンバー（パーティー）を指定して作成
- クエストに紐づく **アイデア** の登録・管理（一覧・編集・削除など基本 CRUD）
- アイデアへの **賛成 / 反対の投票**（参加メンバーによる）
- アイデア単位の **チャットグループ**（メンバー間の意見交換）
- アイデアの **評価**（評価者が 5 観点を 5 点満点で採点＋観点別コメント）
- ゲーミフィケーション要素
  - 経験値（XP）の獲得とレベルアップ
  - コインの獲得
  - ショップでの装備購入
  - 3D アバターへの装備反映
- ユーザー登録・ログイン（ログインでも XP 獲得）
- 企業（テナント）単位の管理
  - 企業ごとにユーザー・アイデアを分離（マルチテナント）
  - 管理者による従業員ユーザーの管理
  - 管理者向けのアイデア閲覧・集計

### やらないこと（Out of Scope）

- （TBD: 例）多人数リアルタイム共同編集、SNS 連携 等 → 要判断

## 4. ユーザー / ペルソナ

| ペルソナ | 概要 | 主なニーズ |
| --- | --- | --- |
| 一般従業員（パーティーメンバー） | クエストに参加してアイデアを投稿・投票・議論する主役ユーザー | 楽しく手軽にアイデアを出したい・仲間と議論したい・成長やアバターを楽しみたい |
| クエスト作成者（オーナー） | クエストを立ち上げ、テーマ・期限・参加メンバーを設定する | 特定テーマでアイデアを集めたい、進行を管理したい |
| 評価者 | アイデアを 5 観点で採点し、観点別コメントを付ける | 出そろったアイデアを公平に評価・選定したい |
| 管理者（企業担当者） | 自社のユーザー・クエスト・アイデアを管理する | アイデアを集約・活用したい、利用状況を把握したい、メンバーを管理したい |
| （TBD）運営 / システム管理 | サービス提供側の全体管理 | 企業（テナント）の管理、アイテムの管理 等 |

> 補足: 「評価者」はグローバルな固定ロールではなく、**クエスト単位で決まる権限**。
> クエスト作成者は常に評価者となり、加えて作成者が任意のユーザーへ評価者権限を付与できる（第6節フロー詳細参照）。

## 5. 主要ユースケース / ユーザーストーリー

- [ ] クエスト作成者として、カテゴリー・テーマ・期限・参加メンバーを決めてクエストを立ち上げたい。テーマを絞ってアイデアを集めたいから
- [ ] パーティーメンバーとして、参加中のクエストのテーマに沿ったアイデアを登録したい。仲間と成果を出したいから
- [ ] メンバーとして、他メンバーのアイデアに賛成／反対を投票したい。良いアイデアを見極めたいから
- [ ] メンバーとして、アイデアごとのチャットで議論したい。認識をすり合わせてブラッシュアップしたいから
- [ ] 評価者として、各アイデアを 5 観点（新規性・影響度・実現度・適合性・コスト）で採点しコメントしたい。公平に選定したいから
- [ ] ユーザーとして、思いついたアイデアを素早く登録したい。登録すると経験値がもらえて楽しいから
- [ ] ユーザーとして、毎日ログインして経験値を貯めたい。継続の動機になるから
- [ ] ユーザーとして、貯めたコインで装備を買い、3D アバターを着せ替えたい。自分らしくカスタマイズしたいから
- [ ] ユーザーとして、レベルが上がる実感を得たい。成長を感じたいから

### クエスト → アイデア → 評価 のフロー詳細

全体の流れ:

```
クエスト作成（テーマ・期限・パーティー）
   → メンバーがアイデア投稿
   → メンバーが賛成/反対を投票 ＋ アイデア別チャットで議論
   → 評価者が各アイデアを 5 観点で採点・コメント
   → 結果を集計・選定
```

#### クエスト内の権限モデル

権限は**クエスト単位**で管理し、対象は**そのクエストの参加メンバー（パーティー）に限定**する（＝権限を持つには参加メンバーであることが前提）。1 メンバーは複数の権限を併せ持てる。

| 権限 | できること | 付与できる人 | 備考 |
| --- | --- | --- | --- |
| 所有者権限 | クエストの**全操作**（下記すべてを含む） | **クエスト作成者のみ**が他ユーザーへ付与できる | クエスト作成者は既定で所有者。**作成者の所有者権限は解除不可**。他ユーザーへの付与により所有者は複数存在しうる |
| クエスト管理権限 | クエストの編集（参加メンバーの追加/削除、各権限の付与/剥奪 を含む） | 所有者 / クエスト管理権限者 | 所有者権限の付与だけは作成者のみ（上記） |
| 評価者権限 | アイデアに対する評価（採点・コメント）の入力 | 所有者 / クエスト管理権限者 | — |
| アイデア投票権限 | アイデアへの投票（賛成/反対） | 所有者 / クエスト管理権限者 | — |
| アイデア作成権限 | アイデアの作成 | 所有者 / クエスト管理権限者 | — |
| コメント作成権限 | アイデアに紐づくグループチャットでのコメント | 所有者 / クエスト管理権限者 | — |

補足・ルール:
- 所有者権限は他のすべての権限を包含する（所有者は評価・投票・作成・コメント・管理すべて可能）。
- **所有者権限の他ユーザーへの付与はクエスト作成者のみ**が行える。その他の権限（管理/評価/投票/作成/コメント）は、所有者またはクエスト管理権限者が付与・剥奪できる。
- クエスト作成者の所有者権限は剥奪・委譲による喪失ができない（常に所有者）。
- 権限付与の対象は参加メンバーのみ。メンバーをパーティーから外すと、そのクエストで保持していた権限も失う（＝以降は投票/評価/作成/コメント/管理ができない）。ただし**そのメンバーがそれまでに入力した内容（アイデア・投票・評価・コメント）は削除せず、表示されたまま残す**。
- **新規参加メンバーの既定権限**: パーティー参加時に **アイデア投票権限・アイデア作成権限・コメント作成権限** を自動付与する。所有者/クエスト管理/評価者の各権限は既定では付与せず、必要に応じて所有者・管理権限者が付与する。

#### 投票

- 対象: アイデア投票権限を持つ参加メンバー（パーティーメンバー）
- 選択肢: **賛成 / 反対**（1人1票、締切まで変更可能を想定）
- **自分のアイデアへの投票も許可**する
- **記名投票**: 誰が賛成/反対したかを公開する
- 表示: アイデアごとに、**賛成したユーザーのアバター** と **反対したユーザーのアバター** をそれぞれ並べて表示する（賛成数・反対数もあわせて表示）
- **XP**: 投票したユーザーに XP を付与する（付与量は TBD）。ただし **評価者による評価には影響しない**（投票結果と評価スコアは独立）

#### チャット

- アイデアごとに 1 つのチャットグループを自動作成
- 参加者はそのアイデアの関係メンバー（TBD: パーティー全員か、限定か）
- メッセージ投稿・履歴表示（TBD: ファイル添付、メンション、通知）

#### 評価（評価者による採点）

1 つのアイデアに対し、評価者が以下 5 観点を **各 5 点満点** で採点し、**観点ごとにコメント** を記載する。

| 観点 | 説明 | 配点 |
| --- | --- | --- |
| 新規性 | アイデアの目新しさ・独自性 | 1〜5 |
| 影響度 | 実現した場合のインパクトの大きさ | 1〜5 |
| 実現度 | 実現可能性の高さ | 1〜5 |
| 適合性 | クエストのテーマ・目的への合致度 | 1〜5 |
| コスト | コスト面の妥当性（TBD: 低コストほど高得点か、要定義） | 1〜5 |

- 各観点に採点値 + コメント（テキスト）を保持
- 合計/平均点でアイデアを比較・ランキング（TBD: 重み付けの有無）
- **評価者（決定）**: クエストごとに以下のユーザーが評価できる。
  - **クエスト作成者**（デフォルトで評価者）
  - **クエスト作成者が評価者権限を付与したユーザー**（クエスト単位で付与・剥奪）
  - → 評価者は 1 クエストに複数存在しうる（＝1アイデアに複数評価者の評価が付きうる）
- 評価者権限を付与できる対象は **そのクエストの参加メンバーに限定（決定）**。詳細は上記「クエスト内の権限モデル」を参照

## 9. データモデル

（主要エンティティの案。詳細は別ファイル `データモデル.md` に分割予定）

- **Company / Tenant**: 企業（テナント）。ユーザー・クエスト・アイデアの所属先
- **User**: ユーザー。所属企業、ロール（一般／評価者／管理者）、XP、レベル、コイン残高を保持
- **Quest**: クエスト。所属企業、作成者（`owner_id`＝既定で所有者・剥奪不可）、カテゴリー、目的/テーマ、期限日、ステータス、作成日時
- **QuestMember（パーティー）**: クエスト参加メンバー（User × Quest の多対多、参加日時を保持）。権限を持つ前提となる所属
- **QuestMemberPermission**: クエスト参加メンバーが持つ権限（QuestMember に対し多）。権限種別（所有者/クエスト管理/評価者/投票/アイデア作成/コメント）、付与者、付与日時。※作成者の所有者権限は剥奪不可
- **Idea**: アイデア。所属クエスト、所属企業、投稿ユーザー、件名、本文（アイデア）、タイムリミット、利害関係者、価値、備考、作成日時
- **Attachment**: アイデアの添付ファイル（Idea に対し多）。ファイル名、参照、アップロード者
- **Vote**: アイデアへの投票（User × Idea）。種別（賛成／反対）、投票日時。1人1票（ユニーク制約）。記名（投票者を表示）。自分のアイデアへの投票も可
- **ChatGroup**: アイデアごとのチャットグループ（Idea と 1:1）
- **ChatMessage**: チャット投稿。所属 ChatGroup、投稿者、本文、投稿日時（TBD: 添付）
- **Evaluation**: アイデアへの評価（Idea × 評価者）。評価者、評価日時
- **EvaluationScore**: 評価の観点別スコア（Evaluation に対し多）。観点（新規性/影響度/実現度/適合性/コスト）、点数（1〜5）、コメント
- **Item / Equipment**: 装備アイテム。名称、種別、価格、3Dモデル参照
- **UserItem**: ユーザーが所有・装備中のアイテム（多対多）
- **Transaction / Activity**: XP・コインの付与・消費履歴（不正防止・監査用、案）

## 13. 用語集

| 用語 | 定義 |
| --- | --- |
| ideaquest | 本プロジェクトの名称 |
| クエスト | アイデア出しのテーマ単位。カテゴリー・目的/テーマ・期限・参加メンバーを持つ |
| パーティー / パーティーメンバー | クエストに参加するメンバーの集まり・各メンバー |
| 投票 | アイデアに対する賛成／反対の意思表示 |
| チャットグループ | アイデアごとに作られる、メンバー間の意見交換の場 |
| 評価者 | アイデアを 5 観点で採点・コメントする役割のユーザー（評価者権限を持つ参加メンバー） |
| 評価観点 | 新規性・影響度・実現度・適合性・コストの 5 項目（各 5 点満点） |
| 所有者権限 | クエストの全操作が可能な権限。作成者は既定で所有者（剥奪不可）。他者への付与は作成者のみ |
| クエスト管理権限 | クエストの編集（参加メンバー調整・権限付与含む）が可能な権限 |
| XP（経験値） | 行動に応じて付与されるポイント。累積でレベルが上がる |
| レベル | XP 累積に応じたユーザーの成長度合い |
| コイン | ショップで装備を購入するための通貨 |
| 装備 / アイテム | 3D アバターに着せ替えできる要素 |
| アバター | ユーザーを表す 3D キャラクター |
